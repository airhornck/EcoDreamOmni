# BUG-2026-0605-003：创建向导 `/generate/create` — Mode C 合规 + 2项功能缺失

> **修复日期**: 2026-06-05（Bug1）/ 待评审（Bug2、Bug3）
> **严重级别**: 🟡 P1 — 功能缺陷 + 需求偏差
> **影响范围**: `/generate/create` 创建向导页面
> **修复者**: Kimi Code CLI
> **状态**: 🟡 部分修复（Bug1 已修复，Bug2/Bug3 触发 PRD 缺失，待评审）

## 一、现象描述（3个关联 Issue）

### Issue 1: 工作区存在「取消」按钮（Mode C 违规）

`/generate/create` 页面工作区画布底部存在一个「取消」按钮：

```tsx
<Button variant="ghost" onClick={() => navigate('/task-hub')}>
  <X className="w-4 h-4 mr-1" />
  取消
</Button>
```

该按钮与右侧 Copilot 面板中的「取消」Quick Action 功能重复，且放置在工作区画布上，违反 Mode C 红线。

### Issue 2: Step 2 缺少关键词选择

用户在 Step 2（人设与故事）中无法选择关键词。当前 Step 2 仅有：
- Persona 选择（必填）
- PersonaStory 选择（可选）
- 故事节点选择（可选）

无关键词输入/选择控件。

### Issue 3: Step 3 Agent 选择后缺少内容模板推荐

用户在 Step 3（Agent 选择）中：
- ✅ 可以看到 Agent 卡片网格
- ✅ 有「已智能推荐」Badge
- ❌ 选完 Agent 后，看不到内容模板（ContentTemplate）的推荐或手动选择

---

## 二、复现条件

1. 登录系统，访问 `http://localhost:5173/generate/create`
2. **Issue 1**: 页面底部可见「取消」按钮
3. **Issue 2**: 点击 Step 2，页面无人设/故事/节点之外的选择控件
4. **Issue 3**: 点击 Step 3，选完 Agent 后无模板推荐区域

---

## 三、根因分析

### Issue 1 根因 — Mode C 实现遗漏

`TaskHubCreatePage.tsx` 在 Mode C 改造时移除了「上一步/下一步/确认创建」按钮，但遗漏了底部的「取消」按钮。Copilot 面板中已提供「取消」Quick Action，工作区按钮 redundant。

### Issue 2 根因 — PRD 未定义

**PRD v4.0 真源检查**:
- `EcoDream_Omni_PRD_v4_AI_Native_Architecture.md` §5.4: 4步向导定义为「选账号→配主题→设风格→确认生成」
- `后端需求补充_内容生产_Copilot-Driven_2026-06-05.md` §1.1: Step 2 定义为「人设与故事」
- **无任何 PRD 章节定义 Step 2 需要关键词选择**
- 关键词功能仅在 **Playground（实验室）** 页面定义（爆款笔记分析 → 关键词推荐）

关键词在 Pipeline 中的角色：`PlatformContentTypeStyle.recommended_keywords` 是 **Step 5 自动生成注入**，非用户创建任务时手动选择。

### Issue 3 根因 — PRD 未定义

**PRD v4.0 真源检查**:
- PRD 中 `MarketingMethodology` Agent 执行「结构模板推荐」，但定义为 **Pipeline Step 2（执行阶段）**，非创建向导 Step 3
- `ContentTemplate` 管理在 Playground 页面独立定义
- **无任何 PRD 章节定义 Step 3 需要内容模板推荐/选择**

## 六、修复方案

### Issue 1 — 直接修复（Bug）

移除 `TaskHubCreatePage.tsx` 底部的「取消」Button，保留提示文本。

```diff
- <Button variant="ghost" onClick={() => navigate('/task-hub')}>
-   <X className="w-4 h-4 mr-1" />
-   取消
- </Button>
  <p className="text-sm text-muted-foreground">
    配置预览模式 — 请在右侧 Copilot 面板中确认创建
  </p>
```

### Issue 2 — 暂停，需 PRD 补充 + 专家评审

**问题**: Step 2 添加关键词选择是**新增功能**，PRD 中无定义。

**需要决策的问题**:
1. 关键词来源：从 `keyword_library` 表（1929条）中选择？还是自由输入？
2. 选择方式：多选下拉框？标签式选择？搜索+勾选？
3. 与 Pipeline 的关联：用户选择的关键词如何影响 Step 5 的 `keyword_inject`？
4. 数据模型：`tasks` 表是否需要新增 `selected_keywords` 字段？
5. 后端 API：是否需要 `GET /api/keywords?platform=` 接口？

**触发专家评审条件**: ✅ 修改核心交互范式（Step 2 增加新控件）+ 可能新增数据模型字段

### Issue 3 — 暂停，需 PRD 补充 + 专家评审

**问题**: Step 3 添加内容模板推荐是**新增功能**，PRD 中无定义。

**需要决策的问题**:
1. 推荐逻辑：基于 Agent + Platform + Format 匹配模板？还是基于内容主题？
2. 模板来源：`content_templates` 表（已有）还是 `structure_definitions`？
3. 用户可选：推荐 Top3 + 手动浏览全部？还是强制选择？
4. 与 Agent 的关系：选模板后是否影响 Agent 的 Prompt？
5. 数据模型：`tasks` 表是否需要新增 `content_template_id` 字段？

**触发专家评审条件**: ✅ 修改核心交互范式 + 可能新增数据模型字段
