1. 业务概述

EcoDreamOmni 是宠物健康素人号矩阵 AI 内容管理与分发平台。

  • 核心能力：AI 辅助内容生成 + 自动化发布 + 流量预测 + 合规风控
  • 目标平台：小红书、抖音、视频号
  • 技术底座：hermes-agent（AI 层）+ openclaw（SaaS 层）
  • 合规红线：100% 拦截处方药/诊断/治疗承诺；商业内容强制标注"合作/体验"

3. 用户操作流程

步骤 1：账号与素材准备

  1. 进入 账号池 /account-pool，批量导入素人账号
  2. 进入 人设池 /personas，创建/选择素人人设模板
  3. 进入 LLM 驾驶舱 /llm-cockpit，配置模型路由和 API 密钥
  4. 进入 平台规则 /platform-rules，同步各平台最新风控规则

步骤 2：创建内容任务

  1. 进入 任务中心 /task-hub，点击"新建任务"
  2. 选择工作流模板（标准 8-Phase 或自定义模板）
  3. 填写 Prompt 变量（关键词、人设、平台、发布时间等）
  4. 绑定发布计划（选择 CronHub 定时任务或即时发布）
  5. 提交任务，系统自动分配 Agent 执行

步骤 3：工作流自动执行

  1. TrendScout 捕获热点情报 → 生成选题报告
  2. ContentForge 基于 AIPL 模板生成内容草稿
  3. PoolPredictor 预测互动量区间（低/中/高三档）
  4. ComplianceGuard 执行 L1-L4 四层合规审核
  5. 系统推送审核通知至运营人员

步骤 4：人工审核与发布

  1. 进入 人工审核台 /human-in-the-loop
  2. 查看待审核内容列表，逐项审核
  3. 操作：通过 / 驳回 / 打回修改
  4. 高风险内容触发双人复核机制
  5. 审核通过的内容进入 发布队列
  6. CronHub 按设定时间执行多平台分发

步骤 5：数据回流与优化

  1. T+24h 系统自动导入平台数据
  2. 进入 数据报表 /data-analyst，查看昨日战报
  3. 查看 MAPE 精度评估和归因分析
  4. 触发异步校准流程，优化预测模型
  5. 查看 Prompt 追踪，定位内容效果差异根因
  6. 基于数据反馈，回到 Phase 1 优化下一轮内容策略

5. 专家评审意见

5.1 业务流程评审

✅ 通过 — 8-Phase 业务流程设计完整，覆盖了从选题洞察到数据回流的完整闭环。合规审核（L1-L4）和人工审核（强制）的串联设计满足风控要求。

5.2 用户操作流程评审

✅ 通过 — 5 步用户操作流程逻辑清晰，从账号准备到数据优化形成完整闭环。各步骤之间的衔接通过系统通知和状态流转实现，减少用户记忆负担。
⚠️ 建议改进 — 当前人工审核台（Human-in-the-Loop）与合规审核（ComplianceGuard）之间的状态流转需要更明确的可视化，建议增加"审核进度条"组件。

5.3 功能框架评审

✅ 通过 — 五大能力域划分合理，模块粒度适中。17+ 核心模块覆盖了 PRD v5.0 定义的全部功能需求。
⚠️ 建议改进 — 部分模块（如 TrendScout、PoolPredictor）的前端页面仍为占位状态，建议按优先级分批实现：
  1. P0：任务中心、内容锻造、合规审核、发布管理（核心业务流程）
  2. P1：趋势侦察、互动预演、数据报表、人工审核台（数据驱动）
  3. P2：账号池、人设池、平台规则、系统设置（运营支撑）

5.4 导航体系评审

✅ 通过 — Sidebar 按 6 大分组重新设计（内容生产 / 风控与发布 / 数据智能 / Agent 治理 / 编排与调度 / 系统配置），与 PRD 业务流程对齐。所有 18+ 路由均已注册，跳转无 404。
⚠️ 建议改进 — Dashboard 快捷操作和快捷导航的 8 个入口已对齐核心业务流程，但建议增加"最近访问"或"收藏功能"的个性化快捷入口。

5.5 合规红线检查

✅ 通过 — 当前实现满足以下红线要求：
  • 禁止展示流量池层级 L0-L5（前端无此展示）
  • 禁止 CES 精确值作为唯一成功口径（DataAnalyst 使用区间描述）
  • interval_mode=prior 须展示「参考区间」（PoolPredictor API 已支持）
  • 优化建议须为清单式启发项（ComplianceGuard 返回清单式建议）
