## 二、关键发现：架构层级不一致

### 2.1 问题描述

PRD V3.1 明确定义了三层架构：
```
Function层（基础功能/数据真源）
    ├── AssetPool、BrandKnowledge、VetDrugDB、TimelineLibrary、PlatformRule
    ├── LLM Hub、CronHub、TaskHub、Workflow Engine、Prompt Registry
    └── AgentHub、AgentWatch、AgentMetrics

Agent层（业务决策）
    ├── TrendScout、ContentForge、ComplianceGuard、Publisher、DataAnalyst
    ├── PoolPredictor、MarketingMethodology、ImageForge、CommentMonitor
    └── 共9个业务Agent

Skill层（原子能力）
    └── 各类可被编排的原子Skill
```

**但开发计划 v2.7.1（第1591行起）存在以下偏差**：

| 偏差项 | 开发计划v2.7.1描述 | PRD V3.1要求 | 严重程度 |
|--------|-------------------|-------------|---------|
| AssetPool定位 | "三源混合素材库"作为独立模块开发 | 应为**Function层**，数据真源 | 🔴 P0 |
| BrandKnowledge定位 | "企业知识库"作为Agent开发 | 应为**Function层**，RAG真源 | 🔴 P0 |
| TimelineLibrary定位 | "时间线库"与CronHub集成 | 应为**Function层**，营销时间真源 | 🔴 P0 |
| VetDrugDB缺失 | 完全未提及兽药批文库 | V3.1新增**Function层**核心组件 | 🔴 P0 |
| W14基础功能周 | 未设置 | V3.1要求**W14为基础功能建设周** | 🔴 P0 |
| Agent直接操作数据 | 未明确禁止 | V3.1红线：Agent**禁止直接操作数据库** | 🔴 P0 |

### 2.2 根因分析

1. **开发计划 v2.7.1 基于 V2.7.1 早期版本编写**，未同步 V3.1 的架构层级调整
2. **五大基础功能被当作业务模块**，未识别其"数据真源"的基础设施属性
3. **缺少W14基础功能先行周**，导致Agent开发缺乏数据基座

## 三、评审决议与修正方案

### 3.1 架构层级重新定义

**决议**: 开发计划必须采用 PRD V3.1 的三层架构定义

```
Layer 3: Function层（基础功能/数据真源）- W14优先建设
├── AssetPool Function（图库/素材库真源）
├── BrandKnowledge Function（品牌知识真源）
├── VetDrugDB Function（兽药批文真源）- V3.1新增
├── TimelineLibrary Function（营销时间真源）
├── PlatformRule Function（平台规则真源）
├── LLM Hub / CronHub / TaskHub / Workflow / Prompt Registry / Human-in-the-Loop
└── AgentHub / AgentWatch / AgentMetrics

Layer 2: Agent层（业务决策）- W15-W17接入Function
├── TrendScout Agent（调用TimelineLibrary）
├── ContentForge Agent（调用BrandKnowledge/VetDrugDB）
├── ComplianceGuard Agent（调用BrandKnowledge/VetDrugDB/PlatformRule/AssetPool）
├── ImageForge Agent（调用AssetPool/BrandKnowledge）- V2.7.1新增
├── CommentMonitor Agent（调用BrandKnowledge/PlatformRule）- V2.7.1新增
└── 现有Agent升级接入Function

Layer 1: Skill层（原子能力）- 通过Agent调用Function
└── 各类Skill（部分允许直接调用Function API白名单）
```

### 3.2 周次调整方案

| 原计划 | 调整后 | 变更说明 |
|--------|--------|---------|
| W15: TrendScout+5A+BrandKnowledge | **W14: 五大基础功能建设** | 新增基础功能先行周 |
| W16: AssetPool+ImageForge+... | **W15: Agent接入基础功能** | TrendScout/ContentForge/ComplianceGuard接入 |

| W18: E2E验证 | **W18: 全链路E2E+安全审计** | 20账号×标准工作流×五大基础功能验证 |

## 五、评审结论

### 5.1 一致性检查

| 检查项 | 结果 | 说明 |
|--------|------|------|
| PRD V3.1三层架构完全采纳 | ✅ | 开发计划必须按Function/Agent/Skill分层 |
| 五大基础功能作为数据真源 | ✅ | W14优先建设，Agent禁止直接操作数据库 |
| VetDrugDB兽药批文库纳入 | ✅ | 合规刚需，作为独立Function |
| W14基础功能先行周设立 | ✅ | 为Agent开发提供数据基座 |
| 文档引用关系明确 | ✅ | PRD为产品真源，开发计划为技术实施真源 |

### 5.2 执行决议

1. **开发计划 v2.7.1 必须按本评审报告进行架构对齐更新**
2. **W14基础功能建设周为新增的强制性里程碑**，不可跳过
3. **所有Agent/Skill对基础功能的访问必须通过Function API**，禁止直接操作数据库
4. **VetDrugDB兽药批文库为V3.1新增核心Function**，必须纳入开发计划
5. **更新后的开发计划版本号建议为 v2.7.1-V3.1对齐版**
