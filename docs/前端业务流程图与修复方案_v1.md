## 四、核心闭环页面实现方案

### 4.1 ContentCreatePage (最核心)
采用Demo的5步向导模式，对接真实API：

```
Step 1: 任务配置
  - API: GET /account-pool, GET /personas, GET /methodologies
  - 表单: 任务名称、账号、人设、AIPL阶段、Topic、风格
  
Step 2: AI编排 (Pipeline)
  - API: POST /pipeline/tasks 或 POST /content-generate
  - 模拟进度展示
  
Step 3: 人工审核
  - 展示: 生成内容、合规结果、预测区间
  - API: POST /human-in-the-loop/tasks/{id}/approve|reject|revise
  
Step 4: 发布调度
  - API: POST /publish-tasks
  - 选项: 立即发布 / 定时发布
  
Step 5: 完成
  - 导航到任务中心或数据分析
```

### 4.2 TaskHubPage (任务中心)
- API: GET /task-hub/tasks
- 功能: 任务列表、状态筛选、搜索
- 状态流转: draft → reviewing → approved → published

### 4.3 PublisherPage (发布管理)
- API: GET /publish-tasks, POST /publish-tasks/{id}/execute
- 功能: 待发布列表、已发布列表、执行发布

### 4.4 CompliancePage (合规中心)
- API: GET /compliance/rules, POST /compliance/check
- 功能: 合规规则列表、内容合规检查、审核记录

### 4.5 DataAnalystPage (数据报表)
- API: GET /data-analyst/dashboard, POST /data-analyst/reports
- 功能: 昨日战报、归因分析、CSV导入

### 4.6 AssetPoolPage (素材库)
- API: GET /assets, POST /assets
- 功能: 素材CRUD、上传、分类
