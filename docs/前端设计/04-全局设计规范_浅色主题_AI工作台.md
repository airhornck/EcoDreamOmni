## 一、产品定位与设计哲学

### 1.1 一句话定位

> **EcoDream Omni 是 AI-Native 的内容生产与矩阵运营工作台** —— 让运营团队像指挥 AI 舰队一样，批量生产、审核、发布内容，管理矩阵账号。

### 1.2 设计哲学

| 原则 | 含义 | 落地方式 |
|------|------|---------|
| **AI First** | AI 不是附属功能，是核心交互方式 | 每个页面都有 AI Copilot 入口，AI 能感知上下文 |
| **画布思维** | 内容生产是创作过程，不是填表 | 中央大面积画布，内容可视化编辑 |
| **渐进披露** | 不一次性展示所有功能 | 默认简洁，按需展开高级功能 |
| **温暖专业** | 不冰冷、不玩具感 | 暖白底色 + 柔和紫强调 + 圆润卡片 |
| **效率优先** | 减少点击、减少等待 | Command+K 全局命令、批量操作、键盘快捷键 |

### 1.3 与竞品的差异化

| 维度 | Notion | ChatGPT | Gemini | **EcoDream Omni** |
|------|--------|---------|--------|------------------|
| 核心交互 | 文档块 | 对话流 | 对话流 | **画布 + AI Copilot** |
| 内容形态 | 文档 | 文本 | 多模态 | **内容卡片 + 数据面板** |
| AI 角色 | 辅助写作 | 问答助手 | 通用助手 | **运营副驾驶（Copilot）** |
| 组织方式 | 页面树 | 对话历史 | 对话历史 | **工作台画布 + 项目看板** |
| 强调色 | 柔和多彩 | 品牌绿 | 渐变流光 | **紫罗兰渐变** |

## 三、全局布局架构

### 3.1 三栏持久布局

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│  ┌──────────┐  ┌──────────────────────────────────────────┐  ┌──────────┐  │
│  │          │  │                                          │  │          │  │
│  │ IconNav  │  │          Workspace Canvas                │  │ AI       │  │
│  │  (52px)  │  │          (自适应宽度)                     │  │ Copilot  │  │
│  │          │  │                                          │  │ (320px)  │  │
│  │          │  │  ┌────────────────────────────────────┐  │  │          │  │
│  │          │  │  │        Bento Grid / 画布            │  │  │          │  │
│  │          │  │  │        内容卡片 + 数据面板          │  │  │          │  │
│  │          │  │  └────────────────────────────────────┘  │  │          │  │
│  │          │  │                                          │  │          │  │
│  └──────────┘  └──────────────────────────────────────────┘  └──────────┘  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

#### 左栏：Icon Nav（52px 宽）

- 顶部：Logo（E 字母图标）
- 中部：主功能图标（12 项）
- 底部：用户头像（点击展开账户菜单）
- 激活态：左侧 3px 紫条 + 图标紫色
- Hover：图标文字 tooltip

#### 中栏：Workspace Canvas（自适应）

- 顶部：Page Header（面包屑 + 标题 + 操作区）
- 主体：根据页面类型渲染不同画布
- 底部：留白区域（滚动到底部时 48px 留白）

#### 右栏：AI Copilot Panel（320px 宽，可折叠）

- 顶部：面板标题 + 折叠按钮
- 中部：对话历史 + Action Cards
- 底部：输入框 + 快捷操作

### 3.2 页面类型体系

| 页面类型 | 特征 | 代表模块 |
|---------|------|---------|
| **工作台** | Bento Grid + 数据面板 | Dashboard、TaskHub |
| **画布** | 中央大面积编辑区 | ContentForge、实验室 |
| **看板** | Kanban 列 + 卡片拖拽 | TaskHub（项目视图） |
| **列表** | 表格/列表 + 筛选栏 | AccountPool、ReviewPublish |
| **详情** | 左右分栏（信息 + 编辑）| PersonaDetail、TaskDetail |
| **数据** | 图表 + 指标卡片 | DataAnalyst、TrendScout |
| **设置** | 表单 + 开关 | Settings、PlatformRules |

## 五、AI Copilot 设计规范

### 5.1 定位

> AI Copilot 不是聊天机器人，而是**上下文感知的运营副驾驶** —— 它知道你当前在看什么页面、选中了什么内容，能主动提供操作建议。

### 5.2 面板结构

```
┌─────────────────────────────┐
│  🤖 AI Copilot        [—]   │  ← 标题栏（可折叠）
├─────────────────────────────┤
│                             │
│  💡 当前任务：              │  ← Context Bar
│  "猫咪驱虫避坑指南"         │
│  [优化标题] [生成配图]      │  ← Action Chips
│                             │
├─────────────────────────────┤
│                             │
│  👤 这篇内容结构不错        │  ← 对话历史
│  🤖 建议加入具体数字...     │
│                             │
├─────────────────────────────┤
│  [🎨] [📎] [🎙️]            │  ← 工具栏
│  ┌───────────────────────┐  │
│  │ 告诉 AI 你的需求...    │  │  ← 输入框
│  │                 [➤]   │  │
│  └───────────────────────┘  │
└─────────────────────────────┘
```

### 5.3 核心交互模式

#### 模式 1：上下文感知（Context-Aware）

当用户在不同页面时，Copilot 自动感知上下文：

| 用户所在页面 | Copilot 上下文 | 主动建议 |
|-------------|---------------|---------|
| TaskHub | 选中任务列表 | "批量生成这 3 个任务的内容？" |
| ContentForge | 编辑内容 #123 | "优化这段标题的点击率？" |
| DataAnalyst | 查看趋势图 | "这个下降趋势需要关注" |
| AccountPool | 选中账号 | "这个账号 3 天未发布，要补发吗？" |

#### 模式 2：Action Cards

AI 返回的操作卡片，用户一键执行：

```
┌─────────────────────────────────────┐
│  🤖 建议优化标题                    │
│                                     │
│  原标题：猫咪驱虫避坑指南           │
│  优化后：猫咪驱虫避坑指南，这3个    │
│          误区90%的人都不知道        │
│                                     │
│  [应用修改] [再优化] [忽略]         │
└─────────────────────────────────────┘
```

#### 模式 3：Inline AI（行内 AI）

在内容编辑区域，选中文字后浮现 AI 操作：

```
选中的文字
     ┌─────────────────────┐
     │ ✨ 优化  |  🔄 改写   │
     │ 🌐 翻译  |  📏 扩写   │
     └─────────────────────┘
```

#### 模式 4：Agent 状态看板

Copilot 面板顶部显示当前运行的 Agent：

```
🟢 TrendScout  已完成  0.8s
🟢 ContentForge 进行中  ⏱ 8.5s
⚪ Compliance   等待中
```

### 5.4 视觉规范

| 元素 | 样式 |
|------|------|
| 用户消息 | 右对齐，浅灰底 `#F3F4F6`，圆角 14px |
| AI 消息 | 左对齐，白底，左侧 3px 紫条 `#7C3AED` |
| Action Card | 紫渐变边框（流动动画），白底 |
| 输入框 | 底部固定，带附件/语音/图片按钮 |
| 流式输出 | 打字机效果 + 紫色光标闪烁 |

## 六、工作台画布 Workspace Canvas

> 工作台不是仪表板（Dashboard），而是**画布** —— 用户在上面排列、组合、操作内容卡片，像 Figma 一样自由，但像 Notion 一样简单。

## 八、交互与动效规范

### 8.1 微动效原则

| 原则 | 说明 |
|------|------|
| **有目的** | 每个动效都服务于反馈或引导 |
| **有节制** | 动画时长 150~300ms，不拖沓 |
| **有质感** | 使用 ease-out 曲线，避免 linear |

### 8.2 标准动效

| 交互 | 时长 | 曲线 | 效果 |
|------|------|------|------|
| 按钮 Hover | 150ms | `ease-out` | 背景色变 + 微上移 |
| 卡片 Hover | 200ms | `ease-out` | 阴影加深 + 上移 2px |
| 页面切换 | 250ms | `ease-in-out` | 淡入 + 微上移 8px |
| 弹窗打开 | 200ms | `cubic-bezier(0.16, 1, 0.3, 1)` | 缩放 0.95→1 + 淡入 |
| 弹窗关闭 | 150ms | `ease-in` | 淡出 |
| Copilot 消息 | 300ms | `ease-out` | 从底部滑入 |
| AI 流式输出 | 逐字 | `linear` | 打字机效果 |
| 卡片拖拽 | 实时 | — | 半透明 + 阴影放大 |
| 加载骨架 | 1.5s | `linear` | Shimmer 流光 |

### 8.3 特殊动效

#### AI 流光边框

用于 AI 生成中的卡片：
```css
border: 2px solid transparent;
background: linear-gradient(white, white) padding-box,
            linear-gradient(90deg, #7C3AED, #A78BFA, #7C3AED) border-box;
background-size: 200% 100%;
animation: flow 2s linear infinite;
```

#### 脉冲指示器

Agent 运行中状态：
```css
.pulse-dot {
  animation: pulse 2s ease-in-out infinite;
  box-shadow: 0 0 0 0 rgba(124,58,237,0.4);
}
```
