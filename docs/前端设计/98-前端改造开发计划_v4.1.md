### 2.2 AI 原生体验设计原则

| 原则 | 定义 | 落地方式 |
|------|------|---------|
| **感知即服务** | AI 感知用户当前上下文，主动提供服务 | Copilot ContextBar 随页面/选中内容变化 |
| **建议即操作** | AI 建议不只是文字，是可一键执行的操作 | Action Cards 带 [应用] [确认] 按钮 |
| **生成即协作** | AI 生成内容时，用户可实时干预 | 六层 Prompt 可视化 + 流式输出可暂停 |
| **反馈即学习** | 用户每次操作都反馈给 AI，持续优化 | 内容评分、Agent Trace 时间线 |
| **无处不在** | 每个页面都有 AI 入口，不是独立功能 | Inline AI 浮层 + Copilot 持久面板 |

### 1.4 AI Copilot 面板增强（Day 5~8）

#### 1.4.1 上下文感知系统（Context-Aware）

**当前问题**: Copilot 不知道用户在看什么页面、选中了什么内容

**实现方案**:
```typescript
// stores/aiCopilotStore.ts — 扩展
interface CopilotContext {
  page: string              // 当前页面路由
  pageTitle: string         // 当前页面标题
  selectedItems: string[]   // 选中的内容/任务 ID
  selectedContent?: {       // 选中的内容摘要
    title: string
    content: string
    platformId: string
  }
  activeTask?: string       // 当前活跃任务
}

// 页面组件中上报上下文
useEffect(() => {
  setContext({
    page: '/generate',
    pageTitle: '内容生产',
    selectedItems: selectedTaskIds,
    selectedContent: currentTask ? {
      title: currentTask.title,
      content: currentTask.content,
      platformId: currentTask.platformId,
    } : undefined,
  })
}, [selectedTaskIds, currentTask])
```

**主动建议规则**（对齐设计规范 §5.3）:

| 用户所在页面 | 选中内容 | 主动建议 |
|-------------|---------|---------|

| `/` 工作台 | 选中 3 个任务卡片 | "批量生成这 3 个任务的内容？" |
| `/generate` | 编辑内容 #123 | "优化这段标题的点击率？" |
| `/review` | 选中待审内容 | "这篇合规分数偏低，要我帮你修改吗？" |

#### 4.3.3 各 Tab 详细设计

**Tab 1: 笔记编辑（NoteEditorZone）**
- 标题输入框（单行文本）
- 正文编辑器（textarea + 简易 Markdown 预览）
- emoji 快捷输入面板（P2 引入 `emoji-mart`）
- 封面拖拽上传区（无上传按钮，拖拽即触发）
- 互动数据输入（点赞/收藏/评论，数字输入框，可选）
- 赛道标签选择（下拉框，来源现有分类库）

**Tab 2: 分析预览（AnalysisPreviewZone）** ⭐ 核心
- 左侧：原文展示，关键词高亮（5 色标注不同维度）
- 右侧分析摘要卡片：
  - 结构类型标签（大字号彩色标签 + 置信度）
  - 爆款评分（SVG 环形进度条）
  - 情绪曲线（recharts 迷你折线图）
  - Top3 成功因子（列表）

**Tab 3: 报告详情（ReportDetailZone）**
- 可折叠 JSON 树（自研递归组件，默认 depth ≥ 2 折叠）
- 各部分分析详情卡片（标题/开头/正文/结尾/emoji/标签）
- 关键词匹配表格（位置、权重、功能）— **Admin 角色可见**

**Tab 4: 模板预览（TemplatePreviewZone）**
- 公式可视化（正则 `/{(.*?)}/g` 分片 + 彩色 Badge 映射）
- 约束条件列表（只读标签展示）
- 自检规则列表（通过/未通过状态标签）
- JSON 语法高亮（纯 CSS 高亮或 `prismjs`）

#### 4.3.6 性能优化重点

| 风险点 | 应对方案 |
|--------|---------|
| 关键词高亮渲染阻塞（>3000字，>50处匹配）| Web Worker 预计算高亮索引；可视区外延迟渲染；按维度筛选展示 |
| JSON 树 DOM 膨胀 | 默认折叠 depth ≥ 2；大数据数组截断（前 10 + `...and N more`）；React.memo |
| 情绪曲线重绘 | useMemo 缓存图表数据；Tab 切换用 `display: none` 保留图表实例 |
| Markdown 编辑器大文本 | 内容长度上限 10000 字；useDeferredValue 解耦输入与预览 |
