# 前端改造开发提示词（分阶段推进版）

> **使用场景**: 每次进入新 Phase、每日开发开始时、任务卡壳时调用
> **输入**: 当前 Phase 编号 + 页面名称（如 Phase1-主题重构 / Phase2-工作台）
> **输出**: 该任务的完整上下文 + 执行步骤 + 验收标准 + 关联文档索引
> **关联文档**: `98-前端改造开发计划_v4.1.md`、`04-全局设计规范_浅色主题_AI工作台.md`

---

## 提示词正文（复制使用）

```
你正在推进 EcoDream Omni v4.0 前端改造任务。

【当前任务上下文】
- Phase: {phase_number}（Phase0-基础设施 / Phase1-P0核心 / Phase2-P1支撑 / Phase3-P2P3优化）
- 页面: {page_name}（如 工作台/内容生产/审核发布/...）
- 路由: {route_path}（如 /generate）
- 页面类型: {page_type}（BentoGrid/看板/列表/详情/数据面板/设置）
- 开发者: {developer_name}
- 开始日期: {start_date}

【你必须遵循的文档体系】（按优先级读取）
1. 前端开发计划: docs/前端设计/98-前端改造开发计划_v4.1.md
   → 读取当前 Phase 的任务清单、AI 体验触点、里程碑
2. 全局设计规范: docs/前端设计/04-全局设计规范_浅色主题_AI工作台.md
   → 读取该页面的布局定义、设计 Token、动效规范、AI Copilot 规范
3. 前端设计总纲: docs/前端设计/00-前端设计总纲.md
   → 读取页面类型、路由、组件目录
4. 数据词典: docs/数据词典_v4.0/
   → API 端点查 02-后端API路由层.md，Store 映射查 04-前端Store与路由.md
5. 契约与数据: docs/契约与数据/01-API接口契约.md
   → 错误码/枚举/分页/幂等性规范
6. 工程纪律: docs/工程纪律_v4.0.md
   → 检查 6 条架构红线 + 测试覆盖率 + 类型检查
7. 需求真源: EcoDream_Omni_PRD_v2_对齐核心方案.md
   → 需求争议时回来对真源

【Step 0 - 一致性检查】（必须先做，发现冲突先解决）
□ 读取 src/index.css，确认主题色（当前暗色 #0A0B0F，目标浅色 #FFFFFF）
□ 读取 src/App.tsx，确认路由定义与数据词典一致
□ 确认当前 Phase 优先级（P0 未完成禁止开发 P1）
□ 确认 index.css 的 utility classes 可用（glass-panel/card-hover/ai-glow/page-transition）

【Step 1 - HTML 预览构建】（必须先做，审核通过才能进入 Step 3）
□ 文件位置: demo/page-preview/{page-name}.html
□ 使用 Tailwind CDN + index.css 的 Token 定义
□ 必须包含: IconNav(52px) + Workspace Canvas + AI Copilot(320px) 占位
□ 必须包含: AI Copilot 上下文感知示例（选中内容后面板变化）
□ 必须包含: Command+K 入口、所有交互状态（Hover/Active/Focus/Loading/Empty）
□ 必须包含: 响应式断点（Desktop/Tablet/Mobile）
□ 使用真实中文 mock 数据，禁止空占位符
□ 自查清单:
  □ 布局符合设计规范 §该页面布局定义
  □ 颜色 Token 与 index.css 一致，禁止硬编码色值
  □ 复用 index.css utility classes，禁止重复定义
  □ 所有数据区域有真实 mock 内容
  □ 按钮有 Hover/Active/Focus/Disabled 四态
  □ 卡片有 card-hover 动效
  □ AI Copilot 上下文感知交互有示例
  □ 空状态、加载状态、错误状态均有设计
□ 提交人工审核（产品+设计+技术负责人）

🛑 【强制停止点】HTML 预览审核
   完成 Step 1 后，必须在此处停止并等待人工审核。
   ┌─────────────────────────────────────────────────────────────┐
   │  下一步行动（由人类决策者执行）：                              │
   │  1. 打开 demo/page-preview/{page-name}.html 预览              │
   │  2. 产品负责人确认：功能完整、文案正确、mock 数据真实         │
   │  3. 设计负责人确认：布局/颜色/动效符合设计规范               │
   │  4. 技术负责人确认：Token 复用正确、无重复定义               │
   │  5. 三方全部通过后，回复 "审核通过"，方可进入 Step 2/3       │
   └─────────────────────────────────────────────────────────────┘
   ⚠️ 在收到明确的 "审核通过" 指令前，禁止执行 Step 2 及后续步骤。

【Step 2 - 契约先行】（仅在 Step 1 HTML 预览通过人工审核后执行）
   前置条件: 收到产品+设计+技术负责人明确的 "审核通过" 回复
□ 新增/修改 API 端点 → 更新 docs/契约与数据/01-API接口契约.md
□ 新增/修改枚举值 → 更新 01-API接口契约.md §三
□ 新增/修改错误码 → 更新 01-API接口契约.md §二
□ 新增路由 → 更新 docs/数据词典_v4.0/04-前端Store与路由.md

【Step 3 - 开发实现】（仅在 Step 1 HTML 预览通过人工审核后执行）
   前置条件: 收到产品+设计+技术负责人明确的 "审核通过" 回复
□ 组件复用优先: shadcn/ui 组件 > index.css utility classes > 自行实现
□ Store 与 Query 分工:
  - TanStack Query: 服务端数据获取、缓存、重试
  - Zustand: 全局客户端状态（auth、Copilot、布局）
  - useState: 组件内部状态
□ API 响应必须解包 data.data（后端返回 {code, message, data}）
□ 所有 API 调用通过 Store/Query，禁止组件中直接 fetch
□ 空值零容忍:
  - 加载中 → Skeleton（animate-shimmer）
  - 数据为空 → EmptyState（含 AI 建议文案）
  - 字段缺失 → 显示 '--' 或默认值
  - 枚举未映射 → 显示映射值（xhs→小红书）
□ 类型安全: 0 errors（npx tsc --noEmit）
□ 组件拆分: 页面文件 < 400 行，拆分为子组件
□ AI 原生体验（该页面必须实现的 AI 触点）:
  - 工作台: ContentCard Agent Trace + AI Insight Card
  - 内容生产: 六层 Prompt 可视化 / Inline AI 浮层
  - 审核发布: 合规分数 + AI 修改建议
  - 数据报表: 异常检测 / AI 战报
  - 账号矩阵: 健康度预警 / AI 发布计划
  - Agent 舰队: Agent 状态看板实时更新
  - 模型中心: 成本可视化
□ Copilot 上下文上报: 每个页面 useEffect 中调用 setContext({page, pageTitle, selectedItems, selectedContent})

【Step 4 - 联调测试】
□ API 对齐:
  □ code === 'OK' 或预期成功码
  □ 请求参数与数据字典一致
  □ 响应从 data.data 正确解包
  □ 分页解析 pagination 字段
  □ 错误码处理（400/401/403/404/429/500/503/504）
□ AI 体验:
  □ Copilot 上下文感知（切换页面后 ContextBar 更新）
  □ 选中内容后 Copilot 主动建议
  □ Action Cards 有可执行按钮
  □ 空状态有 AI 建议文案
  □ Agent Trace 正确展示（成功/运行中/失败）
  □ 流式输出有打字机效果
□ 边界测试: 空数据、超长文本、特殊字符、网络错误、慢网络、限流、权限不足
□ 视觉回归: 与 HTML 预览对比、主题一致、动效流畅
□ 性能: 首屏 < 2s、大数据虚拟滚动、图片懒加载、搜索防抖 300ms
□ 后端缺口发现:
  □ 发现 API 缺失/字段缺失/枚举缺失 → 立即记录
  □ 输出后端需求补充文档（docs/后端需求/后端需求补充_{Page}_{日期}.md）
  □ 模板参考: docs/提示词模板/03-后端需求补充文档模板.md
  □ 前端用 mock/workaround 继续开发，不阻塞进度

【Step 5 - 质量门禁】（必须全部通过）
□ TypeScript 0 错误（npx tsc --noEmit）
□ ESLint 0 错误 0 warning（eslint src/ --max-warnings=0）
□ Prettier 格式一致（prettier src/ --check）
□ 测试覆盖率 ≥ 60%（npm test -- --coverage）
□ 空值检查通过（grep 未处理的可空字段）
□ API 包装层检查通过（grep 未解包 response.data.data）
□ 枚举映射检查通过（无原始枚举值直接显示）
□ utility 复用检查通过（无重复定义 index.css 已有 class）
□ 路由一致性检查通过（与 App.tsx 一致）
□ 主题一致性检查通过（与 index.css 一致）
□ AI 体验检查通过（人工审查 + 交互测试）
□ Copilot 上下文检查通过（页面切换后 ContextBar 正确更新）

【Step 6 - 文档连锁更新】
代码完成后，按以下顺序更新文档:
1. docs/前端设计/字段映射_{PageName}.md → Store 字段 ↔ API 字段映射表
2. docs/数据词典_v4.0/04-前端Store与路由.md → 如有新增 Store/路由
3. docs/前端设计/00-前端设计总纲.md → 开发状态标记
4. docs/前端设计/98-前端改造开发计划_v4.1.md → 进度标记
5. docs/契约与数据/01-API接口契约.md → 如有 API/枚举/错误码变更
6. docs/文档总纲_v4.0.md → 状态标记
7. docs/后端需求/ → 如有后端缺口，输出后端需求补充文档

【禁止事项】
- ❌ 跳过一致性检查直接开发
- ❌ 跳过 HTML 预览直接开发
- ❌ HTML 预览未经人工审核通过即进入 Step 2/3
- ❌ index.css 主题不一致时开发新页面
- ❌ 使用 any/unknown 类型（normalize 入参 Record<string, unknown> 除外）
- ❌ 直接从 response.data 取数据（未解包 .data 字段）
- ❌ API 调用用 camelCase（调后端必须用 snake_case）
- ❌ 原始枚举值直接显示（如 xhs 而非 小红书）
- ❌ 组件中直接 fetch/axios（必须通过 Store/Query）
- ❌ 页面文件 > 400 行未拆分
- ❌ 重复定义 index.css 已有 utility class
- ❌ 未更新数据字典/契约文档/开发计划即提交
- ❌ 测试覆盖率 < 60%
- ❌ 未处理 VALIDATION_ERROR/RATE_LIMITED/AGENT_DEGRADED 等错误码
- ❌ 用户输入 dangerouslySetInnerHTML 渲染
- ❌ 页面无 AI 原生体验（未实现该页面要求的 AI 触点）
- ❌ 未上报 Copilot 上下文
- ❌ P0 未完成时开发 P1，或 P1 未完成时开发 P2/P3
- ❌ 发现后端缺口不记录、不输出后端需求补充文档
- ❌ 前端自行修改后端代码绕过缺口
```

### 实验室 · 爆款笔记分析引擎速查

> **前置条件**: C1-C6 全部满足（见 `docs/变更记录_v4.0/2026-06-05/笔记分析功能_专家评审报告.md` §四）
> **工期**: 前端 3 周 + 后端 2 周并行
> **模式**: Mode C（工作区无业务按钮，操作走 Copilot Action Cards）

**组件清单**:
```
components/lab/
├── NoteEditorZone.tsx           # Tab 1: 笔记编辑
├── AnalysisPreviewZone.tsx      # Tab 2: 分析预览（核心）
│   ├── KeywordHighlighter.tsx   # 关键词高亮（5色，性能优化重点）
│   ├── ViralScoreRing.tsx       # 爆款评分环形图（SVG自研）
│   ├── EmotionCurveChart.tsx    # 情绪曲线（recharts）
│   └── SuccessFactorList.tsx    # Top3 成功因子
├── ReportDetailZone.tsx         # Tab 3: 报告详情
│   └── JsonTree.tsx             # 可折叠JSON树
├── TemplatePreviewZone.tsx      # Tab 4: 模板预览
│   ├── FormulaVisualizer.tsx    # 公式可视化
│   └── ConstraintList.tsx       # 约束条件列表
├── hooks/
│   ├── useViralAnalyze.ts       # POST /api/lab/analyze
│   ├── useViralTemplate.ts      # POST /api/lab/template
│   └── usePlaygroundContext.ts  # Copilot上下文上报
```

**Copilot Action Cards 状态机**:
```
empty → input_ready → analyzing → analyzed → template_ready
  │         │            │           │            │
  │    「📤粘贴笔记」  「🔍分析」  「📋查看报告」  「💾保存模板」
  │                        │           │            │
  └────────────────────────┴───────────┴────────────┘
```

**API 端点**:
| 端点 | 用途 |
|------|------|
| POST `/api/lab/analyze` | 爆款笔记分析 |
| POST `/api/lab/template` | 生成模板草稿 |
| GET `/api/lab/keywords` | 关键词库（高亮配色）|
| POST `/api/content-templates` | 保存模板（复用现有）|

**性能注意**:
- 关键词高亮 >3000字+50处匹配时用 Web Worker
- JSON树默认折叠 depth≥2
- 情绪曲线用 useMemo + display:none 保活

**MVP 裁剪**（如工期紧张）:
- ✅ 保留: 笔记编辑Tab、分析预览Tab（高亮+评分）、Copilot Action Cards
- ❌ 延后: 报告详情JSON树→纯文本、模板预览公式可视化→纯文本、情绪曲线→Phase2

## 关键检查命令速查

```bash
# === 一致性检查 ===
grep -n "#0A0B0F\|#FFFFFF" src/index.css          # 主题色确认
grep -o 'path="[^"]*"' src/App.tsx | sort | uniq  # 路由确认

# === 类型与 Lint ===
npx tsc --noEmit
npx eslint src/ --max-warnings=0
npx prettier src/ --check

# === 测试 ===
npm test -- --coverage --watchAll=false

# === 空值与类型 ===
grep -rn ": any\|: unknown" src/ --include="*.ts" --include="*.tsx" | grep -v "Record<string, unknown>"
grep -rn "{[a-zA-Z_][a-zA-Z0-9_]*}" src/pages --include="*.tsx" | grep -v "??" | grep -v "||"

# === API 包装层 ===
grep -rn "response\.data\.map\|response\.data\." src/stores --include="*.ts" | grep -v "response\.data\.data"

# === utility 复用 ===
grep -rn "glass-panel\|card-hover\|page-transition" src/pages --include="*.tsx" | grep -v 'className="'

# === AI 体验 ===
grep -rn "setContext\|useAICopilotStore" src/pages --include="*.tsx"
grep -rln "InlineAI\|ActionCard\|AgentTrace\|SixLayerPrompt\|AIInsight" src/pages --include="*.tsx"
```
