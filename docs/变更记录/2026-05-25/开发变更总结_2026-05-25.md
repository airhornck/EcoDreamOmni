## VF-4: `llm_hub.py` — AsyncSession 解耦

### 目标
`llm_hub.py` 是 LLM 模型注册中心，包含模型 CRUD、Scope 配置、用量日志、成本汇总等约 40 个 DB 访问点。将其从 Service 层降级为纯业务逻辑层，不再持有 `AsyncSession` 类型依赖。

### 修改文件

| 文件 | 变更点 |
|------|--------|
| `apps/backend/src/services/llm_hub.py` | 删除 `from sqlalchemy.ext.asyncio import AsyncSession`；所有函数签名 `db: AsyncSession` → `db: Any`（21 处替换） |

### 保留在 Service 层的纯逻辑

- `encrypt_api_key()` / `decrypt_api_key()` — 加解密逻辑
- `get_provider_region()` — 供应商区域映射
- `_default_endpoint()` — 默认端点计算
- `_calc_cost()` — 成本计算公式

### API 层适配

`api/llm_hub.py` 中所有 `db: AsyncSession = Depends(get_db)` 保持不变，向下传递 `db` 到 `llm_hub.py` 时类型擦除为 `Any`，不影响运行时行为。

## VF-6: `celery_tasks.py` — AsyncSessionLocal 提取到 Function 层

### 目标
`celery_tasks.py` 是 Celery Worker 的任务定义文件，其 `execute_workflow` 任务内部直接 `from src.core.database import AsyncSessionLocal` 并 `async with AsyncSessionLocal() as db:` 驱动 workflow，属于**最严重的架构违规**（Celery worker 绕过了 API 层的 `Depends(get_db)` 生命周期管理）。

此外，该任务还**复制了 `task_hub._resume_workflow()` 和 `task_hub.start_workflow()` 的驱动逻辑**，造成代码重复。

### 新增文件

| 文件 | 说明 |
|------|------|
| `apps/backend/src/services/celery_tasks_function.py` | **Function 层** — 持有 `AsyncSessionLocal` 上下文管理器，封装 `drive_workflow()` 异步函数，统一驱动 workflow 从当前节点执行到完成或 human gate |

### 修改文件

| 文件 | 变更点 |
|------|--------|
| `apps/backend/src/services/celery_tasks.py` | 删除 `from sqlalchemy.ext.asyncio import AsyncSession` 和 `from src.core.database import AsyncSessionLocal`；删除内嵌 `_drive()` 协程；改为 `from src.services.celery_tasks_function import drive_workflow` 并 `asyncio.run(drive_workflow(task_hub_task_id))` |
| `apps/backend/tests/test_no_direct_db_access.py` | `FUNCTION_LAYER_SERVICES` 列表新增 `"celery_tasks_function.py"` |

### 设计说明

```
 celery_tasks.py (Agent/Skill 层)
   └── @shared_task execute_workflow()
         └── import drive_workflow (from Function 层)
               └── async with AsyncSessionLocal() as db:  ← 唯一持有 DB 连接的地方
                     └── task_hub.get_task(db, ...)       ← db: Any 向下传递
                     └── workflow_engine.execute_next_node(...)
```

- `celery_tasks.py` 仅保留 `@shared_task` 装饰器和同步/异步桥接（`asyncio.run`）
- `celery_tasks_function.py` 是 Function 层，独享 `AsyncSessionLocal` 导入权
- 向下调用 `task_hub` / `workflow_engine` 时，`db` 以 `Any` 类型传递，符合架构分层规范
