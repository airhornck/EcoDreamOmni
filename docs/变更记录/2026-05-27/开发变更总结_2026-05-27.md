## 调用关系变更

### B 组

```
前端: PersonasPage.tsx → usePersonaStoryStore → fetch('/api/persona-stories/{id}/nodes/reorder')
后端: persona_story.py (已有端点，未变更)
```

- **删除链路**：`StoryCockpitPage.tsx` → `usePersonaStoryStore` → `/api/persona-stories`、`/api/story-nodes`
- **保留链路**：`PersonasPage.tsx` → `usePersonaStoryStore` → `/api/persona-stories`、`/api/story-nodes`（功能完整覆盖）

### A 组前端

```
前端: PublisherPage.tsx → 无创建入口（action=undefined）
前端: PublisherPage.tsx → 提示横幅 → 路由跳转 /review-publish-center
```

### A 组后端（统一后链路）

```
review_publish.py confirm-publish → publisher_service.create_publish_task(task_hub_task_id=...) → task_hub.transition_task("running")

publisher.py create_task ──→ 强制要求 task_hub_task_id
                         └──→ task_hub.get_task() 校验 APPROVED_WAITING_PUBLISH
                         └──→ publisher_service.create_publish_task(task_hub_task_id=...)
                         └──→ task_hub.transition_task("running")
```

**关键约束**：任何 publish_task 的创建都必须关联已审核通过的 TaskHub task；`/publish-tasks` 不再接受无 `task_hub_task_id` 的请求（返回 403）。

### C 组

```
前端: AgentOrchestraPage.tsx → useAgentCockpitStore →
  ├─ GET /agent-cockpit/dashboard        (fleet status + agent table + alerts + traces)
  ├─ GET /agent-cockpit/alerts/summary   (severity/type/status 聚合)
  ├─ GET /agent-cockpit/activity         (merged task/alert/heartbeat stream)
  ├─ GET /agent-hub/agents               (agent registration list)
  ├─ GET /agent-hub/agents/{id}/configs  (config version history)
  ├─ GET /agent-metrics/overall          (completion rate / tokens / cost)
  ├─ GET /agent-metrics/cost/by-agent    (cost attribution table)
  ├─ GET /agent-watch/alerts             (alert list with ack)
  └─ PATCH /agent-watch/alerts/{id}/ack  (acknowledge alert)
```

**后端已就绪**：`agent_hub.py`、`agent_watch.py`、`agent_metrics.py`、`agent_cockpit.py` 四个模块及对应测试均已实现并通过。
