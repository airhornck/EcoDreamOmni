## 三、晚间 — 决策实施阶段

### 3.1 W16 代码分割优化（决策 3A）

**目标**：将前端主包从 ~1.1MB 降至可接受范围，实现路由级懒加载。

| 文件 | 变更类型 | 说明 |
|------|----------|------|
| `apps/frontend/src/App.tsx` | 修改 | 全部 24+ 页面路由转换为 `React.lazy(() => import(...))` + `Suspense` 包裹 |

**构建结果**：
- 主包 `index-*.js`: **306KB** (gzip 94KB)，原 ~1.1MB
- 14+ 独立 chunk（TaskHubPage 24KB / AccountPoolPage 24KB / ReviewPublishDetailPage 402KB 等）
- 构建时间: ~450ms

## 四、调用关系变更

```
# AccountPool 配额
Publisher 发布成功 → publisher_service.execute_publish → account.posts_today += 1
Workflow Publisher 节点 → task_hub._simulate_node_output → account.posts_today += 1
confirm_publish → evaluate_l3 → account.quota_exceeded check → 阻断 or 通过

# PlatformRule 工作流对接
Workflow compliance-guard 节点 → ORM evaluate_content (L1-L4) + check_compliance fallback → merged compliance_result
Workflow Publisher 前 → evaluate_l3 → ORM L3/L4 rules + memory fallback → allowed/disallowed

# 前端路由
App.tsx → React.lazy() × 24 routes → Suspense fallback → 按需加载 chunk
```
