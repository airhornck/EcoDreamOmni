# 实验室 — 爆款笔记分析引擎 功能设计方案

## 一、功能定位

| 项目 | 说明 |
|------|------|
| **功能名称** | 爆款笔记分析引擎（ViralAnalyzer） |
| **所属模块** | 实验室工作区 |
| **核心目标** | 用户上传爆款笔记 → AI分析结构 → 生成可复用模板 → 供内容生成调用 |
| **功能边界** | **只做分析+模板生成，不做内容生成**（生成由其他Agent负责） |
| **用户角色** | 内容运营、产品经理、AI训练师 |

## 三、功能流程设计

### 3.1 主流程：单次分析

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  上传笔记  │───→│  解析输入  │───→│  关键词匹配 │───→│  结构分析  │───→│  输出报告  │
│          │    │          │    │          │    │          │    │          │
│ 用户操作  │    │ 前端处理  │    │ AI Copilot│    │ AI Copilot│    │ 画布展示  │
│ [上传按钮]│    │ 文本清洗  │    │ 调用分析Skill│   │ 生成结构化  │    │ 报告+模板  │
│          │    │ 分词切分  │    │          │    │ JSON报告   │    │          │
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘
                                                                         │
                                                                         ▼
                                                                  ┌──────────┐
                                                                  │  生成模板  │
                                                                  │          │
                                                                  │ 用户确认后 │
                                                                  │ 保存到模板库│
                                                                  └──────────┘
```

### 3.2 详细步骤

| 步骤 | 触发方式 | 执行方 | 输入 | 输出 | 依赖 |
|------|---------|--------|------|------|------|
| **S1: 上传笔记** | 点击AI Copilot【上传笔记】按钮 | 前端 | 用户粘贴/上传的笔记数据 | 结构化表单（标题、正文、封面、互动数据、赛道标签） | 无 |
| **S2: 解析输入** | 自动触发 | 前端 | 原始笔记数据 | 清洗后的结构化数据 | **公共依赖：分词服务** |
| **S3: 关键词匹配** | 点击【开始分析】按钮 | AI Copilot → 后端Skill | 结构化笔记数据 | 关键词匹配结果（结构/功能/情感/行业/效果关键词） | **公共依赖：关键词库** |
| **S4: 结构分析** | S3完成后自动触发 | 后端Skill | 关键词匹配结果 | 结构化分析报告JSON | **公共依赖：结构定义库** |
| **S5: 输出报告** | S4完成后自动触发 | 前端 | 分析报告JSON | 画布区可视化展示 | 无 |
| **S6: 生成模板** | 点击【生成模板】按钮 | AI Copilot → 后端Skill | 分析报告JSON | 标准化模板JSON | **公共依赖：模板Schema定义** |
| **S7: 保存模板** | 点击【保存模板】按钮 | 后端 | 模板JSON + 用户确认 | 存储到模板库 | **公共依赖：模板存储服务** |

## 四、数据模型

### 4.1 输入数据（用户上传）

```typescript
// 笔记输入结构
interface NoteInput {
  title: string;                    // 笔记标题
  content: string;                  // 正文内容（Markdown）
  coverImage?: string;              // 封面图片URL/Base64
  metrics?: {                       // 互动数据（可选）
    likes: number;
    collects: number;
    comments: number;
  };
  category: string;                 // 赛道标签（下拉选择）
  tags?: string[];                  // 话题标签
}
```

### 4.2 分析输出（Layer 1）

```typescript
// 分析报告结构
interface AnalysisReport {
  noteId: string;                   // 笔记唯一ID
  structureType: StructureType;     // 结构类型枚举
  viralScore: number;               // 爆款评分 0-100
  
  // 关键词匹配结果
  keywordMatches: {
    structure: KeywordMatch[];      // 结构关键词匹配
    function: KeywordMatch[];       // 功能关键词匹配
    emotion: EmotionMatch[];        // 情感关键词匹配
    industry: KeywordMatch[];       // 行业关键词匹配
    effect: KeywordMatch[];         // 效果关键词匹配
  };
  
  // 各部分分析
  titleAnalysis: TitleAnalysis;
  hookAnalysis: HookAnalysis;
  bodyAnalysis: BodyAnalysis;
  ctaAnalysis: CtaAnalysis;
  emojiAnalysis: EmojiAnalysis;
  
  // 情绪曲线
  emotionCurve: EmotionPoint[];
  
  // 成功因子
  successFactors: string[];
  
  // 提取的原始模板（用于下一步生成标准化模板）
  extractedTemplate: ExtractedTemplate;
}

type StructureType = 
  | "种草测评型" 
  | "干货合集型" 
  | "避坑排雷型" 
  | "教程攻略型" 
  | "对比测评型" 
  | "个人故事型";

interface KeywordMatch {
  keyword: string;
  position: number;                 // 在文本中的位置
  weight: number;                   // 权重得分
  function: string;                 // 功能描述
}

interface EmotionMatch extends KeywordMatch {
  emotionType: "焦虑" | "惊喜" | "愤怒" | "向往" | "共鸣" | "信任";
  intensity: number;                // 强度 0-1
}
```

### 4.3 模板输出（Layer 2）

```typescript
// 标准化模板结构
interface ViralTemplate {
  templateId: string;               // 模板唯一ID
  version: string;                  // 版本号
  name: string;                     // 模板名称（用户可编辑）
  
  // 溯源信息
  derivedFrom: string[];            // 来源笔记ID列表
  category: string;                 // 赛道分类
  structureType: StructureType;     // 结构类型
  
  // 关键词公式（核心）
  keywordFormulas: {
    title: KeywordFormula;          // 标题公式
    hook: KeywordFormula;           // 开头钩子公式
    body: BodyFormula;              // 正文公式
    cta: KeywordFormula;            // 结尾CTA公式
  };
  
  // 约束条件（生成时校验）
  constraints: TemplateConstraints;
  
  // 自检规则
  validationRules: ValidationRule[];
  
  // 元数据
  meta: {
    createdAt: string;
    updatedAt: string;
    usageCount: number;
    viralScoreThreshold: number;
  };
}

interface KeywordFormula {
  pattern: string;                  // 公式模式，如 "{数字}个{痛点}，{结果}让我{情绪}"
  requiredKeywords: string[];       // 必须包含的关键词类型
  optionalKeywords: string[];       // 可选关键词类型
  example: string;                  // 示例
}

interface TemplateConstraints {
  titleLength: [number, number];    // 标题字数范围
  bodySectionMin: number;           // 正文最少段落数
  emojiDensity: string;             // emoji密度，如 "2-3/100字"
  hookLength: [number, number];     // 开头字数范围
}
```

### 5.2 按钮详细定义

| 按钮 | 触发时机 | 调用后端 | 前端反馈 | 异常处理 |
|------|---------|---------|---------|---------|
| **上传笔记** | 用户点击 | 无（纯前端） | 弹出表单，填充后画布展示 | 表单校验失败提示 |
| **开始分析** | 用户点击，笔记数据已填充 | `POST /api/viral/analyze` | 显示进度条，关键词高亮动画 | 分析失败显示错误信息 |
| **查看报告** | 分析完成后自动展开/用户点击 | 无（已缓存） | 画布区切换到报告视图 | - |
| **生成模板** | 用户点击，报告已生成 | `POST /api/viral/template` | 显示模板JSON预览 | 生成失败提示 |
| **保存模板** | 用户点击，模板已生成 | `POST /api/template/save` | 保存成功Toast + 模板ID | 保存失败提示 |
| **导出模板** | 用户点击，模板已生成 | 无（前端下载） | 下载JSON文件 | - |
| **清空画布** | 用户点击 | 无 | 确认弹窗 → 清空所有状态 | - |
| **历史记录** | 用户点击 | `GET /api/viral/history` | 侧边栏展开历史列表 | - |

### 7.2 核心分析流程（后端）

```
┌─────────────────────────────────────────────────────────────┐
│                    /api/viral/analyze                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 接收 NoteInput                                          │
│     └── 参数校验（title/content/category必填）                │

│  3. 关键词匹配（核心）                                       │

│     ├── 正文分析：段落切分 + 功能关键词标记                   │
│     ├── 结尾分析：最后100字CTA关键词匹配                      │

│                                                             │
│  7. 爆款评分计算                                            │
│     ├── 结构完整性得分（30%）                                 │
│     ├── 关键词丰富度得分（25%）                               │
│     ├── 情绪曲线得分（20%）                                   │
│     ├── 互动数据加权（15%）                                   │
│     └── emoji策略得分（10%）                                  │
│                                                             │
│  8. 提取原始模板                                            │
│     ├── 基于分析结果组装公式                                  │
│     └── 标记必需/可选关键词                                   │
│                                                             │
│  9. 返回 AnalysisReport                                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 八、公共依赖标注

### 8.1 必须的平台公共功能

| 依赖名称 | 类型 | 用途 | 是否必须 | 备注 |
|---------|------|------|---------|------|
| **分词服务** | 公共服务 | 中文分词、词性标注 | ✅ 必须 | 建议jieba或HanLP，支持自定义词典 |
| **关键词库** | 数据资产 | 6大维度关键词定义 | ✅ 必须 | 需要运营团队持续维护 |
| **结构定义库** | 数据资产 | 6大爆款结构定义 | ✅ 必须 | 相对稳定，季度更新 |
| **赛道分类库** | 数据资产 | 小红书赛道分类 | ✅ 必须 | 与平台内容分类对齐 |
| **模板Schema定义** | 规范标准 | 模板标准化格式 | ✅ 必须 | 定义模板数据结构 |
| **模板存储服务** | 基础设施 | 模板持久化存储 | ✅ 必须 | 可用对象存储或数据库 |
| **用户认证服务** | 基础设施 | 识别当前用户 | ⚠️ 建议有 | 用于历史记录隔离 |

### 8.2 可选的平台公共功能

| 依赖名称 | 类型 | 用途 | 是否必须 | 备注 |
|---------|------|------|---------|------|
| **模板市场服务** | 增值功能 | 模板共享、推荐 | ❌ 可选 | Phase 2再接入 |
| **效果追踪服务** | 增值功能 | 模板使用→实际爆款率 | ❌ 可选 | 需要与发布系统打通 |
| **封面OCR服务** | 增值功能 | 提取封面文字 | ❌ 可选 | 无封面时可跳过 |
| **情感分析模型** | AI模型 | 更精准的情绪识别 | ❌ 可选 | 关键词匹配已够用 |

### 8.3 本功能自建组件

| 组件 | 说明 | 复杂度 |
|------|------|--------|
| ViralAnalyzer Skill | 核心分析逻辑 | 中 |

## 十一、关键设计原则总结

| 原则 | 体现 |
|------|------|
| **单一职责** | 本功能只做分析+模板生成，不做内容生成 |
| **分层解耦** | 分析层 → 模板层 分离，各自独立迭代 |
| **关键词驱动** | 不存储固定模板，存储关键词组合规则 |
| **渐进式加载** | Skill按需激活，不常驻内存 |
| **真源可追溯** | 每个模板记录来源笔记ID |
| **平台依赖最小化** | 明确标注哪些是公共依赖，哪些自建 |
