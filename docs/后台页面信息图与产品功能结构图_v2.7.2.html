UE 设计要点 · PersonaStory：
1. 时间轴编辑器采用纵向卡片流，左侧为节点序号+状态圆点，右侧为节点详情卡片。拖拽手柄位于卡片左上角。

交互规范（Interaction Guidelines）

全局交互组件规范，确保后台各页面体验一致。所有组件遵循「 antd / Arco Design 」设计体系语义。

表单规范（Forms）
*必填字段（红色星号 + 失焦校验）
输入框高度 32px，圆角 6px，错误状态边框变红 + 下方红色提示文案
选填选填字段（灰色提示文字）
Placeholder 格式示例：「输入任务名称，如：猫咪驱虫避坑指南」
长文本输入
Markdown 编辑器采用双栏分屏（左编辑 / 右预览），支持快捷插入变量标签
批量操作
表格行首 Checkbox → 顶部批量操作栏浮现（淡入 200ms）→ 支持「全选当前页/全选所有」
状态与反馈（States & Feedback）
加载状态
按钮内 Spinner（尺寸 14px）+ 文字变「生成中...」；页面级加载用 Skeleton 骨架屏
成功反馈
Toast 通知（右上角，3s 自动消失）+ 关键操作（如发布）用 Modal 二次确认
异常反馈
Inline 错误（表单字段下方）→ 页面级错误（Empty State 插图 + 重试按钮）→ 系统级错误（全屏错误页 + 联系支持）
空状态（Empty State）
插画 + 文案说明 + 主操作按钮（如「暂无任务，去创建一个」）
数据表格（Tables）
默认展示
每页 20 条，支持分页器 + 「加载更多」双模式切换
排序与筛选
表头点击排序（三角箭头）→ 筛选栏折叠在表格上方，默认收起
行操作
主操作外露（编辑/查看详情），次要操作收进「更多」下拉菜单
行内编辑
仅基础功能配置页支持行内编辑（单击文本变输入框，失焦自动保存）
弹窗与抽屉（Modal & Drawer）
Modal 对话框
用于「二次确认 / 轻量表单」，宽度 480px，点击遮罩可关闭（除高危操作外）
Drawer 抽屉
用于「详情查看 / 新建任务 / 配置编辑」，宽度 640px，从右侧滑入（300ms ease-out）
页面跳转 vs 抽屉
规则：若操作后需要「浏览器刷新不丢失上下文」，用页面；若操作上下文在当前页内闭环，用抽屉
全局状态标签色板（Status Badges）
草稿执行中人工审核中已完成失败
未开始进行中已发布暂停
所有状态标签统一使用圆角 pill 形状，禁止在标签内使用图标（保持信息密度）。
响应式断点（Responsive Breakpoints）
