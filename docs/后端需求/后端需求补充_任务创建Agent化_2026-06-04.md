## 一、变更概述

将任务创建流程从「用户选择工作流模板」升级为「用户选择 Agent」。工作流模板降级为 Agent 内部实现细节，不再暴露给终端用户。

**架构原则**：
- Task 永远绑定 `agent_id`
- `workflow_template_id` 保留用于存量兼容，但标记为 deprecated
- Agent 自行决定使用哪些 Skill 和工作流节点

**脚本 2：存量 Task 回填 agent_id（需结合 platform + content_format 二次路由）**

> 由于一个 YAML 模板可能对应多个平台-specific Agent，回填时必须结合任务的 `platform` 和 `content_format` 字段。

```sql

CREATE OR REPLACE VIEW workflow_to_agent_mapping AS
SELECT 
  platform, content_format, workflow_template_id, agent_id
FROM (VALUES
  -- content_creation_note_image (图文笔记)
  ('xiaohongshu', '图文', 'content_creation_note_image', 'content_forge_xhs_image'),
  ('wechat_channels', '图文', 'content_creation_note_image', 'content_forge_wx_text'),

  ('xiaohongshu', '视频', 'content_creation_video_original', 'content_forge_xhs_video'),
  ('douyin', '视频', 'content_creation_video_original', 'content_forge_douyin_video'),
  ('wechat_channels', '视频', 'content_creation_video_original', 'content_forge_wx_video'),
  ('bilibili', '视频', 'content_creation_video_original', 'content_forge_bili_video'),
  
  -- content_creation_video_clone (视频复刻)
  ('douyin', '视频复刻', 'content_creation_video_clone', 'content_forge_douyin_clone'),
  ('bilibili', '视频复刻', 'content_creation_video_clone', 'content_forge_bili_clone'),
  
  -- content_creation_text_article (长文章)
  ('xiaohongshu', '仅文字', 'content_creation_text_article', 'content_forge_xhs_text'),

  -- content_creation_standard / content_creation_light (通用)
  ('xiaohongshu', '图文', 'content_creation_standard', 'content_forge_xhs_image'),
  ('xiaohongshu', '视频', 'content_creation_standard', 'content_forge_xhs_video'),
  ('douyin', '视频', 'content_creation_standard', 'content_forge_douyin_video'),
  ('douyin', '视频复刻', 'content_creation_standard', 'content_forge_douyin_clone'),
  ('bilibili', '视频', 'content_creation_standard', 'content_forge_bili_video'),
  ('bilibili', '视频复刻', 'content_creation_standard', 'content_forge_bili_clone'),
  ('wechat_channels', '图文', 'content_creation_standard', 'content_forge_wx_text'),
  ('wechat_channels', '视频', 'content_creation_standard', 'content_forge_wx_video')
) AS t(platform, content_format, workflow_template_id, agent_id);

-- 执行回填
UPDATE tasks t
SET agent_id = m.agent_id
FROM workflow_to_agent_mapping m
WHERE t.workflow_template_id = m.workflow_template_id
  AND t.platform = m.platform
  AND t.content_format = m.content_format
  AND t.agent_id IS NULL;

-- 兜底：未匹配到的任务使用通用 Agent
UPDATE tasks
SET agent_id = 'content_forge_generic'
WHERE agent_id IS NULL AND workflow_template_id IS NOT NULL;

-- 输出未匹配记录供人工核查
SELECT id, name, workflow_template_id, platform, content_format
FROM tasks
WHERE agent_id IS NULL;
```

**推荐算法（v4.0 确认版，含 Persona 历史偏好权重）**：

```python
def recommend_agent(platform: str, format: str, persona_id: str = None, account_id: str = None):
    """
    综合评分 = 基础分 × Persona 偏好加成 × 负载均衡因子
    """
    # Step 1: 精确匹配（硬筛选）
    candidates = db.agents.filter(
        status='ACTIVE',
        supported_platforms__contains=platform,
        supported_formats__contains=format
    )
    if not candidates:
        return fallback_to_generic_agent(platform, format)
    
    # Step 2: 计算综合评分
    scored = []
    for agent in candidates:
        score = 0.0
        
        # 2.1 成功率权重 (40%)
        score += agent.success_rate * 0.40
        
        # 2.2 Persona 历史偏好权重 (30%)
        if persona_id:
            persona_history = db.task_history.filter(
                persona_id=persona_id,
                agent_id=agent.id,
                created_at__gte=now() - timedelta(days=30)
            )
            if persona_history.count() > 0:
                persona_success_rate = persona_history.success_count / persona_history.count()
                # 该 Persona 用此 Agent 次数越多、成功率越高，权重越大
                familiarity_boost = min(persona_history.count() / 10, 1.0)  # 最多 10 次达到满加成
                score += persona_success_rate * 0.30 * familiarity_boost
            else:
                # 无历史记录时，给基础分避免歧视新 Agent
                score += 0.15  # 中性偏好
        else:
            score += 0.15
        
        # 2.3 负载均衡因子 (20%)
        # recent_tasks_1h 越低，得分越高（反向衰减）
        load_score = max(0, 1.0 - (agent.recent_tasks_1h / agent.max_concurrent_tasks))
        score += load_score * 0.20
        
        # 2.4 账号历史权重 (10%)
        if account_id:
            account_history = db.task_history.filter(
                account_id=account_id,
                agent_id=agent.id,
                created_at__gte=now() - timedelta(days=30)
            )
            if account_history.count() > 0:
                account_success_rate = account_history.success_count / account_history.count()
                score += account_success_rate * 0.10
            else:
                score += 0.05
        else:
            score += 0.05
        
        scored.append((agent, score))
    
    # Step 3: 排序并生成推荐理由
    scored.sort(key=lambda x: x[1], reverse=True)
    top_agent, top_score = scored[0]
    
    # 生成推荐理由
    reason_parts = []
    if persona_id and db.task_history.filter(persona_id=persona_id, agent_id=top_agent.id).count() > 5:
        reason_parts.append(f"该 Persona 过去 30 天使用过 {top_agent.name} {persona_history.count()} 次，成功率 {(persona_success_rate*100):.0f}%")
    else:
        reason_parts.append(f"{top_agent.name} 最近 24 小时成功率 {(top_agent.success_rate*100):.0f}%")
    
    if top_agent.recent_tasks_1h < 5:
        reason_parts.append("当前负载较低，响应更快")
    
    return {
        "recommended_agent_id": top_agent.id,
        "confidence": round(top_score, 2),
        "reason": "；".join(reason_parts),
        "alternatives": [
            {"agent_id": a.id, "name": a.name, "confidence": round(s, 2), "reason": f"成功率 {(a.success_rate*100):.0f}%"}
            for a, s in scored[1:3]
        ],
        "matched_capabilities": list(set(top_agent.skills) & set(required_capabilities))
    }
```

**权重设计 rationale**：
| 因子 | 权重 | 说明 |
|------|------|------|
| 全局成功率 | 40% | Agent 整体质量，最稳定指标 |
| Persona 历史偏好 | 30% | 同一 Persona 用某 Agent 越多次且成功率越高，越推荐（个性化） |
| 负载均衡 | 20% | 避免热点 Agent，保障响应速度 |
| 账号历史 | 10% | 同一账号的历史偏好，弱于 Persona 级别 |

## 四、Agent Harness 改造

### 4.1 ContentForge Agent 内部路由逻辑

ContentForge Agent 不再依赖外部传入的 `workflow_template_id`，而是根据任务参数内部决定 Skill 组合：

```python
class ContentForgeAgent:
    def execute(self, task_context: dict):
        platform = task_context['platform']
        format = task_context['content_format']
        persona_id = task_context['persona_id']
        
        # 1. 根据平台+格式选择 Skill 组合
        skill_chain = self._resolve_skill_chain(platform, format)
        
        # 2. 加载 Agent 配置快照（版本化）
        config = task_context['agent_config_snapshot']
        
        # 3. 执行 Skill 链
        for skill in skill_chain:
            result = self.skill_gateway.invoke(skill, task_context)
            task_context.update(result)
        
        return task_context
    
    def _resolve_skill_chain(self, platform: str, format: str) -> list:
        # 平台+格式 → Skill 组合映射
        chains = {
            ('xiaohongshu', '图文'): [
                'trend_scout', 'marketing_methodology', 
                'brand_knowledge_inject', 'keyword_inject',
                'content_generate', 'image_generate',
                'compliance_check', 'engagement_predict',
                'human_approval', 'publish'
            ],
            ('douyin', '视频'): [
                'trend_scout', 'marketing_methodology',
                'brand_knowledge_inject', 'keyword_inject', 
                'content_generate', 'video_clone',
                'compliance_check', 'engagement_predict',
                'human_approval', 'publish'
            ],
            # ... 其他组合
        }
        return chains.get((platform, format), chains[('xiaohongshu', '图文')])
```

## 八、风险与回滚

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| Agent 映射表不完整 | 存量任务回填失败 | 脚本执行前输出未匹配记录，人工补录 |
| Agent 实例路由故障 | 新任务无法调度 | 保留 workflow_template_id 兼容路径作为 fallback |
| 性能退化 | Agent 推荐查询慢 | GIN 索引 + Redis 缓存热点 Agent 列表 |
| 前端未同步上线 | 新旧接口不一致 | API 层同时接受 agent_id 和 workflow_template_id（二选一） |

**回滚方案**：
1. 数据库：`agent_id` 字段 nullable，可随时清空回退
2. API：保留 `workflow_template_id` 兼容分支，开关控制
3. 代码：Git tag 标记回滚点
