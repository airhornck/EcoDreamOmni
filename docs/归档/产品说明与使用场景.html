用户使用说明

从创建账号到发布内容，五步上手素人号矩阵运营

账号池与人设池配置

1

导入账号矩阵

在 AccountPool 中批量导入小红书账号，配置设备指纹、IP 代理与每日发布配额。系统自动计算账号健康分。

2

创建 Persona（人设）

在 PersonaPool 中定义人设档案：身份背景、语言风格、专业领域、禁忌话题。支持 AI 辅助生成草案。

3

配置 LLM 路由

在 LLM Hub 中为不同 Agent 绑定模型：ContentForge 可用 GPT-4o（创意），ComplianceGuard 强制境内模型（合规）。

AccountPool 配置示例
{
  "account_id": "xhs_001",
  "platform": "xiaohongshu",
  "persona_id": "pet_expert_v1",
  "health_score": 88,
  "status": "active",
  "daily_quota": 3,
  "ip_proxy": "residential_001",
  "fingerprint": { ... }
}

创建任务与选择工作流

1

选择工作流模板

进入 TaskHub，选择预设模板：「标准内容生产」或「轻量快速发布」。MVP 仅支持串行 Pipeline。

2

填写 Prompt 变量

在 Prompt Registry 白名单变量中填入内容参数：{{topic}}、{{style}}、{{target_audience}}。系统自动校验变量合法性。

3

绑定发布计划

选择立即执行、定时发布（CronHub 绑定）或加入队列等待。系统调用 PlatformRule Engine 预检时段合规性。

Prompt 变量白名单示例
{{topic}} STRING · 必填 · 最大100字符
{{style}} ENUM · [口语化,专业,故事]
{{persona_voice}} STRING · 从 PersonaPool 注入

Agent 工作流自动执行

1

TrendScout 热点侦察

系统自动生成趋势报告（Mock 数据源），标注 AIPL 阶段与审核松紧度，推荐选题方向。

2

ContentForge 内容生成

按 MarketingMethodology 模板生成：hook（数字+痛点）→ body（干货结构）→ cta（互动引导）→ disclaimer（合规声明）。

3

PoolPredictor 互动预演

输出点赞/评论/收藏的预测区间（如 25-60/5-15/8-20）与置信度。冷启动阶段展示「参考区间」文案。

4

ComplianceGuard 合规审核

四层规则逐层扫描：法律红线 → 平台规则 → 账号状态 → 动态风控。命中规则则记录证据链，触发拦截或警告。

工作流节点状态实时看板

Human Approval WAITING

人工审核台操作

1

进入审核队列

工作流执行至 Publisher 前强制进入 Human-in-the-Loop。审核台左侧展示图文预览，右侧展示 Agent 输出摘要。

2

三种审核决策

通过：进入发布队列（可选立即/定时）；驳回：任务失败，记录理由；打回修改：指定回退节点并修改变量。

3

双人复核发布

Publisher 发布确认须「内容审核员」与「发布确认员」双角色操作。审核记录自动归档，留存 ≥2 年。

数据回流与模型校准

1

数据导入

发布后 T+24h~T+48h，运营通过 CSV/表单导入实际互动数据（点赞/评论/收藏）。连接器为辅，失败自动降级。

2

昨日战报生成

DataAnalyst 自动聚合：发布篇数、预测区间命中率（覆盖率）、平均互动量、环比变化。CES 仅作可选派生指标。

3

异步模型校准

Celery 批任务检查 MAPE 与漂移。超阈值写入「待重训」队列，由人工或定时任务触发。禁止秒级在线学习承诺。

4

Prompt 效果追踪

Prompt Registry 自动统计各版本的任务完成率、人工干预率、质量评分，辅助运营迭代 Prompt 模板。

技术架构亮点

开源基座 + 自研编排，工程边界清晰

三层 LLM 路由

Global Default → Agent 级 → Skill 级配置粒度，支持固定路由、加权随机与故障转移。国产/国外模型统一纳管，T0/T1/T2 合规分级自动拦截。

四层风控体系

L1 法律红线、L2 平台规则、L3 账号状态（新号/老号差异）、L4 动态风控（节日/时段）。证据链留存 ≥2 年，Publisher 强制双人复核。

全链路可观测

OpenTelemetry 链路追踪、AgentWatch 心跳监控、AgentMetrics 质量评分。人机干预率作为系统健康领先指标，P0/P1/P2 分级告警。

异步校准闭环

数据回流以导入为主，连接器为辅。Celery 批任务检查 MAPE/漂移，超阈值异步触发重训。禁止秒级在线学习承诺，工程边界务实。

版本化治理

AgentHub 配置版本化、Prompt Registry 模板版本化、CronHub Job 版本化三轨对齐。禁止无版本记录热改，支持一键回滚与审计追踪。

工程可靠性声明

99.5% SLA 仅指自建 SaaS API；第三方 LLM、代理、平台站点不计入。回流默认人工导入，覆盖率 KPI 绑定最小样本量（N≥30）。

产品核心能力矩阵

基于 V2.6 PRD 构建的五大能力域，覆盖内容生产、风控合规、数据智能、Agent 治理与任务编排

内容生产引擎

Phase 1
TrendScout

趋势侦察

Mock 数据源 + 手动导入热点，生成结构化趋势报告。支持 AIPL 阶段过滤与人设草案辅助生成，禁止爬虫克隆。

MarketingMethod

方法论中枢

AIPL 四阶段（认知/兴趣/购买/忠诚）定义，每阶段配套 hook/body/cta/disclaimer 内容结构模板与 KPI 目标配置。

ContentForge

内容锻造

Voice 注入式生成，对接 AIPL 阶段模板与合规标签。支持 Prompt 模板变量白名单与版本化回滚。

风控与合规

Phase 1
ComplianceGuard

四层合规审核

L1 法律红线、L2 平台规则、L3 账号状态（新号/老号差异策略）、L4 动态风控（节日/时段策略）。证据链留存 ≥2 年。

Human-in-Loop

人工审核台

发布前强制人工拦截，支持通过/驳回/打回修改。Publisher 发布须双人复核，审核记录全留存，符合《生成式AI服务管理暂行办法》。

PlatformRule

平台规则引擎

L3/L4 动态规则 CRUD，运营可配置日发频率阶梯、时段竞争系数、临时风控策略。违规自动归因分析。

数据智能

Phase 1-2
PoolPredictor

互动量区间预测

冷启动先验宽区间（QuantileRegressor），输出点赞/评论/收藏的 lower/median/upper。禁止 L0-L5 流量池层级预测（已废弃）。

DataAnalyst

数据分析师

24h 回流报表（导入为主），计算区间覆盖率与 MAPE。异步 Celery 校准触发，禁止承诺秒级在线学习。

AccountPool

账号矩阵管理

指纹隔离 + 健康评分 + 自动熔断状态机（warming/active/restricted）。每日配额管控与 IP 代理配置。

Agent 治理体系

Phase 1
AgentHub

配置中心

Agent 注册发现、配置版本化、环境隔离、RBAC 权限、依赖声明。

AgentWatch

活跃监控

30s 心跳、OpenTelemetry 链路追踪、循环/超时/工具失败检测、分级告警。

AgentMetrics

质量统计

任务完成率、Token 成本归因、LLM-as-Judge 质量评分、人机干预率追踪。

LLM Hub

模型路由

国产/国外模型纳管、三层路由（Global/Agent/Skill）、成本限额、熔断降级。

编排与调度

Phase 1
TaskHub

任务中心

任务状态机全生命周期管理（Draft→Queued→Running→HumanWait→Completed），支持批量任务与账号独立队列。

Workflow

工作流引擎

串行 Pipeline 模板（MVP 禁止 DAG），节点级上下文传递（Redis），失败策略可配置（FailFast/Continue/Retry）。

CronHub

定时调度

Cron 表达式调度、分布式锁防重、死信队列（DLQ）、指数退避重试、Dry Run 测试模式。
