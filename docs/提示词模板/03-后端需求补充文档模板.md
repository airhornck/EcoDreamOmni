## 对齐要求

每条需求必须追溯至真源文档：

| 需求类型 | PRD 章节 | 文档总纲章节 |
|---------|---------|-------------|
| 新增 API | PRD 功能模块设计 | `数据词典_v4.0/02-后端API路由层.md` |
| 新增字段 | PRD 数据模型定义 | `数据词典_v4.0/01-数据库模型层.md` |
| 新增枚举 | PRD 状态/类型定义 | `契约与数据/01-API接口契约.md` §3 |
| 新增错误码 | PRD 异常处理设计 | `契约与数据/01-API接口契约.md` §2 |
| SSE/WebSocket | PRD 实时交互设计 | `架构设计/05-EventBus设计.md` |

---

## 协作流程

```
前端测试发现缺口
    → 立即记录到本文档（发现一条记一条）
    → 前端 mock/workaround 继续开发
    → 当前页面测试完成后提交后端评审
    → 后端 24h 内确认排期
    → 后端开发完成 → 自测 → 通知前端
    → 前端移除 workaround → 真实联调
    → 联调通过 → 关闭需求
```
