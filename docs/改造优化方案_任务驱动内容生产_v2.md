### 1.2 核心架构调整

用户审核意见带来的最关键变化：**将「发布确认」从 HumanInLoop 页面中独立出来，新增「审核发布中心」页面。**

**调整后的页面职责划分**：

```
HumanInLoop（人工审核台）          ReviewPublishCenter（审核发布中心）
  ├── 内容预览                        ├── 审核结论列表（已通过/已驳回/已打回）
  ├── Agent 摘要（合规/预演/质量分）   ├── 点击查看审核详情（内容+Agent摘要+审核历史）
  ├── 三种审核决策：                   ├── 对「已通过」任务进行发布确认：
  │   ├── ✅ 通过（带发布模式参数）   │       ├── 立即发布 / 定时发布
  │   ├── ❌ 驳回                     │       └── 循环执行（保存为 CronJob）
  │   └── 🔄 打回修改                 ├── 快捷操作：批量发布 / 批量定时
  └── 决策后任务进入 ReviewPublishCenter  └── 发布后任务进入 Publisher 执行队列
```

**设计理由**：
- **职责分离**：HumanInLoop 专注「审核决策」，ReviewPublishCenter 专注「审核结果查看 + 发布确认 + 循环策略」，符合 PRD 中「单人审核」到「定时发布」的 Phase 划分
- **运营效率**：运营可以批量查看审核结论、批量确认发布，无需逐条进入审核台
- **可追溯性**：审核结论列表形成完整的审核-发布链路记录

## 四、详细改造方案（更新后，含审核发布中心）

### Phase 0：后端数据模型扩展（1d）

#### 4.0.1 Task 状态机扩展

当前状态机：
```
DRAFT → CONFIGURING → QUEUED → RUNNING → PAUSED → HUMAN_WAIT → COMPLETED/FAILED/CANCELLED
```

**新增中间状态**：`APPROVED_WAITING_PUBLISH`

```python
class TaskStatus(str, Enum):
    DRAFT = "DRAFT"
    CONFIGURING = "CONFIGURING"
    QUEUED = "QUEUED"
    RUNNING = "RUNNING"
    PAUSED = "PAUSED"
    HUMAN_WAIT = "HUMAN_WAIT"
    APPROVED_WAITING_PUBLISH = "APPROVED_WAITING_PUBLISH"  # 新增：审核通过，等待发布确认
    COMPLETED = "COMPLETED"
    FAILED = "FAILED"
    CANCELLED = "CANCELLED"

# 状态转换更新
_TRANSITIONS = {
    # ... 已有转换 ...
    TaskStatus.HUMAN_WAIT: {
        TaskStatus.RUNNING,      # 打回修改
        TaskStatus.FAILED,       # 驳回
        TaskStatus.APPROVED_WAITING_PUBLISH,  # 审核通过，进入发布确认队列（新增）
    },
    TaskStatus.APPROVED_WAITING_PUBLISH: {
        TaskStatus.RUNNING,      # 确认发布后执行
        TaskStatus.CANCELLED,    # 取消发布
    },
    # ...
}
```

**设计理由**：
- 将「审核通过」与「确认发布」解耦，运营可以在审核发布中心批量查看已通过的任务，再统一确认发布
- 避免 HumanInLoop 中「通过」即直接发布的耦合设计

```python
@router.post("/review-publish-center/conclusions/{task_id}/confirm-publish")
def confirm_publish(task_id: str, req: ConfirmPublishRequest):
    """审核发布中心：确认发布（立即/定时/循环）"""
    t = task_hub.get_task(task_id)
    if not t or t.status != TaskStatus.APPROVED_WAITING_PUBLISH:
        raise HTTPException(status_code=409, detail="Task is not in APPROVED_WAITING_PUBLISH status")
    
    # 1. 创建发布任务
    publish_task = publisher.create_publish_task(
        draft_id=t.prompt_variables.get("draft_id"),
        account_id=t.account_id,
        platform=t.platform,
        scheduled_at=req.scheduled_at if req.publish_mode == "scheduled" else None,
    )
    
    # 2. 如果需要循环执行，创建 CronJob
    cron_job_id = None
    if req.cron_schedule:
        cron_job = cron_hub.create_job(
            name=f"循环发布: {t.name}",
            schedule=req.cron_schedule,
            target_type="publish_task",
            target_id=publish_task.id,
            concurrency_policy="SKIP",
            owner=req.operator,
        )
        cron_job_id = cron_job.id
    
    # 3. 更新 Task 状态
    task_hub.update_task(
        task_id,
        publish_confirmed_at=_now(),
        publish_confirmer=req.operator,
        cron_job_id=cron_job_id,
    )
    task_hub.transition_task(task_id, "RUNNING")
    
    return {
        "task_id": task_id,
        "status": "RUNNING",
        "publish_mode": req.publish_mode,
        "scheduled_at": req.scheduled_at,
        "cron_job_id": cron_job_id,
    }
```

#### 4.4.3 后端 approve 接口调整

```python
@router.post("/tasks/{task_id}/approve")
def approve_task(task_id: str, req: ApproveRequest):
    """v2 调整：approve 后任务进入 APPROVED_WAITING_PUBLISH，而非直接 RUNNING"""
    t = task_hub.get_task(task_id)
    if not t or t.status != task_hub.TaskStatus.HUMAN_WAIT:
        raise HTTPException(status_code=409)
    
    # v1: 直接 transition 到 RUNNING
    # v2: transition 到 APPROVED_WAITING_PUBLISH
    task_hub.transition_task(task_id, "APPROVED_WAITING_PUBLISH")
    
    # 记录审核决策（但不携带 publish_mode/scheduled_at）
    task_hub.update_task(
        task_id,
        review_decision="APPROVE",
        reviewed_at=_now(),
        reviewer=req.operator,
    )
    
    return {
        "task_id": task_id,
        "status": "APPROVED_WAITING_PUBLISH",
        "message": "审核已通过，请前往审核发布中心确认发布",
    }
```

#### 4.5.5 循环执行（CronJob）集成设计

**复用策略**：审核发布中心不重新实现 CronJob 引擎，直接调用 CronHub 已有 API。

```typescript
// useReviewPublishStore.ts — 发布确认时创建 CronJob

confirmPublish: async (taskId, config) => {
  // 1. 调用审核发布中心确认发布 API
  const res = await fetch(`/api/review-publish-center/conclusions/${taskId}/confirm-publish`, {
    method: 'POST',
    headers: authHeaders(),
    body: JSON.stringify({
      operator: 'operator',
      publish_mode: config.publishMode,
      scheduled_at: config.scheduledAt,
      cron_schedule: config.enableCron ? config.cronSchedule : null,
    }),
  })
  
  if (!res.ok) throw new Error('确认发布失败')
  
  const data = await res.json()
  
  // 2. 如果创建了 CronJob，可选跳转到 CronCockpit 查看
  if (data.cron_job_id) {
    showToast(`已创建循环执行 CronJob，任务 ID: ${data.cron_job_id}`)
  }
  
  return data
}
```

**后端 CronJob 创建逻辑**（已在 Phase 0 的 `confirm_publish` API 中描述）：
- 调用 `cron_hub.create_job()` 创建定时任务
- target_type = "publish_task", target_id = 刚创建的发布任务 ID
- 运营可在 CronCockpit 页面查看和管理已创建的循环任务
