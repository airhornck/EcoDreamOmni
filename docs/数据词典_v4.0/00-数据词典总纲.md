## 二、v4.0 新增数据资产速查

PRD v4.0 在数据层引入了 2 个全新概念和 1 个降级调整：

| 变更 | 类型 | 所属文档 | 说明 |
|------|------|----------|------|
| **PlatformContentTypeStyle** | 全新 ORM | `01-数据库模型层.md` §新增 | 平台内容类型风格指导层，7 个核心字段 |
| **ContentTemplate** | 全新 ORM | `01-数据库模型层.md` §新增 | 爆款复刻模板，8 个核心字段 |
| **DataAnalyst Agent → Function+Skill** | 降级 | `03-后端Service层.md` §更新 | `data_analyst_service` 拆分为 `engagement_collect` Function + `battle_report_generate` Skill |

## 四、各子文档职责

| 文档 | 职责 | 数据来源 | 维护频率 |
|------|------|----------|----------|
| `01-数据库模型层.md` | ORM 字段定义、表关系、索引、迁移记录 | `apps/backend/src/models/` + PRD §7.2 | 每次新增/修改 ORM 时 |
| `02-后端API路由层.md` | 端点路径、Method、请求/响应 Schema、调用 Service | `apps/backend/src/api/` + PRD §6 | 每次新增/修改 API 时 |
| `03-后端Service层.md` | Service 分类、核心方法、调用依赖、事务边界 | `apps/backend/src/services/` + PRD §4 | 每次新增/修改 Service 时 |
| `04-前端Store与路由.md` | Store 定义、Action、Selector、路由表、页面→API 映射 | `apps/frontend/src/stores/` + `src/pages/` | 每次新增/修改页面时 |
| `05-模块调用关系矩阵.md` | API→Service、Service→ORM、Pipeline 节点→Agent/Skill 依赖 | 代码静态分析 + PRD §4.4 | 每次架构变更时 |
| `06-Agent与Skill映射.md` | Agent 清单、Skill 清单、Agent→Skill 调用矩阵、LLM 路由 | PRD §4.2, §4.3 + 代码 | 每次新增/修改 Agent/Skill 时 |

## 五、与 v2.7.x 数据词典的差异

| 维度 | v2.7.x `数据词典/` | v4.0 `数据词典_v4.0/` |
|------|-------------------|----------------------|
| ORM 模型 | 24 个 | 24 + 2 新增（PlatformContentTypeStyle / ContentTemplate） |
| API 端点 | 125 个 | 125 + ~15 新增（实验室 / keyword_inject / PlatformContentTypeStyle / AI Copilot） |
| Agent 映射 | 无独立文档 | 新增 `06-Agent与Skill映射.md` |
| Pipeline 步骤 | 9 步 | 10 步（新增 keyword_inject） |
| DataAnalyst | 作为 Agent | 降级为 Function + Skill |
