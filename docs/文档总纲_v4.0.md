## 七、文档使用指南

### 新成员 Onboarding（v4.0）

1. 阅读根目录 `EcoDream_Omni_PRD_v4_AI_Native_Architecture.md` 了解产品全貌与 8 项架构变更
2. 阅读根目录 `EcoDream_Omni_Architecture_Spec.html` 了解 10 页导航系统与 API 映射
3. 阅读根目录 `EcoDream_Omni_UI_Design_System_v4.md` 了解设计 Token 与组件规范
4. 阅读 `docs/架构设计/00-架构设计总纲.md` 了解 v4.0 新增架构概念
5. 阅读 `docs/数据词典_v4.0/00-数据词典总纲.md` 了解技术架构与模块映射
6. 阅读 `docs/契约与数据/00-契约与数据总纲.md` 了解接口契约与数据模型设计规范
7. 阅读 `docs/工程纪律_v4.0.md` 了解工程纪律与 v4.0 架构红线

### 开发前准备（v4.0）

1. 在 `契约与数据/` 中查找 API 契约、数据库 Schema、状态机定义（设计层）
2. 在 `数据词典_v4.0/` 中查找对应模块的 API/Service/Store/Agent/Skill 映射（实现层）
3. 在 `架构设计/` 中查找对应架构概念的设计决策
4. 在 `前端设计/` 中查找对应页面的 UI 区域与字段定义
5. 参考 PRD v4.0 对应章节，确认需求真源

### 代码审查前（v4.0）

1. 对照 `PRD偏差报告_v4.0.md` 检查是否引入新的偏差
2. 确认是否违反 `docs/工程纪律_v4.0.md` 中的 6 条架构红线
3. 确认 API/Schema/State 变更是否先在 `契约与数据/` 中更新
4. 确认是否同步更新了 `数据词典_v4.0/` 中对应的映射关系

### 文档更新规范

- **新增文档** → 在「有效文档分类清单」中补充一行，同步更新本文档目录结构
- **文档过期** → 移动到 `归档/` 并在「已归档文档」中登记
- **版本升级** → 归档旧版、新增新版、更新本文件
- **v4.0 与 v2.7.x 并行期** → 优先在 v4.0 目录中维护，v2.7.x 文档标记「⚠️ 历史参考」
