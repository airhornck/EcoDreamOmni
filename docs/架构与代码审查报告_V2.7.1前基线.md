## 三、结构性问题识别

### 3.1 🔴 P0 — 模型层缺失（数据持久化危机）

| 指标 | 现状 | 合理值 | 差距 |
|------|------|--------|------|
| ORM模型文件数 | 8 | 25-30 | **缺失~20个模型** |
| 使用内存存储的Service | ~15个 | 0 | **15个服务无持久化** |
| 数据库表（推断） | ~10 | ~35 | **缺失~25张表** |

**使用内存存储的Service清单（需W14-W15迁移）**:
- `agent_hub.py` → `_agent_db: Dict[str, AgentRegistration] = {}`
- `agent_orchestra.py` → `_agent_db`, `_workflow_db`, `_pipeline_db`
- `agent_metrics.py` → 日维度聚合内存计算
- `llm_hub.py` → `_model_db`, `_config_db`（模型注册在内存）
- `task_hub.py` → `_task_db: Dict[str, Task] = {}`
- `asset_pool.py` → `_asset_db: Dict[str, Asset] = {}`
- `cron_hub.py` → `_job_db: Dict[str, CronJob] = {}`
- `prompt_registry.py` → `_template_db: Dict[str, PromptTemplate] = {}`
- `workflow_engine.py` → 执行状态部分内存
- `human_in_loop.py` → `_review_db: Dict[str, ReviewRecord] = {}`
- `platform_rule_service.py` → 规则内存缓存
- `compliance_engine.py` → 规则缓存
- `meta_learner.py` → **已删除**

**影响**: 
- 服务重启=数据丢失
- Phase 3多租户/多实例部署不可能
- 与PRD §2.6 "证据链留存≥2年"直接冲突
