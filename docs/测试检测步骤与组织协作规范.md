## 二、递进式六层测试法

> **核心原则**：下层不通过，上层不开始。每层结束后出具「通过/有条件通过/阻塞」结论。

```
L1 环境连通 ──► L2 认证授权 ──► L3 独立模块 ──► L4 跨模块流程 ──► L5 边界异常 ──► L6 UI/UX一致性
   (10min)        (15min)         (60min/page)      (30min/流程)       (20min/page)       (15min/page)
```

## 三、L1 — 环境连通性检查（技术专家主责）

**目标**：确认前后端、数据库、代理服务全部可访问。
**通过标准**：以下 5 项全部返回预期结果。
**阻塞影响**：L1 不通过则停止所有后续测试。

### 3.1 检查清单

| # | 检查项 | 命令/操作 | 预期结果 | 实际结果 | 通过 |
|---|--------|----------|----------|----------|------|
| 1 | 前端服务 | `curl http://localhost:5173` | HTTP 200 或返回 index.html | | □ |
| 2 | 后端服务 | `curl http://localhost:8000/health` | `{"status":"ok"}` | | □ |
| 3 | 数据库 | `psql -U ecodream -d ecodream -c "SELECT 1"` | 返回 `1` | | □ |
| 4 | Redis | `redis-cli ping` | 返回 `PONG` | | □ |
| 5 | API 文档 | `curl http://localhost:8000/docs` | 返回 Swagger UI HTML | | □ |

### 3.2 常见故障与修复

| 故障现象 | 根因 | 修复方 | 预估时间 |
|---------|------|--------|---------|
| 前端 404 | nginx 未启动 / 端口冲突 | 前端开发 | 5min |
| 后端 502 | Uvicorn 未启动 / 依赖缺失 | 技术专家 | 10min |
| 数据库拒绝连接 | PostgreSQL 未启动 / 密码错误 | 技术专家 | 5min |
| /docs 空白 | main.py 路由注册失败 | 后端开发 | 15min |

## 四、L2 — 认证授权检查（技术专家主责）

**目标**：确认登录/登出/Token/MFA 全链路正常。
**通过标准**：以下 4 项全部通过。
**阻塞影响**：L2 不通过则所有需认证页面无法测试。

### 4.1 检查清单

| # | 检查项 | 操作步骤 | 预期结果 | 实际结果 | 通过 |
|---|--------|---------|----------|----------|------|
| 1 | 登录成功 | 输入正确账号密码 → 点击登录 | 跳转 Dashboard，LocalStorage 有 token | | □ |
| 2 | 登录失败 | 输入错误密码 | 显示错误提示，不跳转 | | □ |
| 3 | Token 过期 | 等待 token 过期 / 手动清除 → 刷新页面 | 自动跳转登录页或弹出重新登录 | | □ |
| 4 | MFA（如启用） | 登录后输入 6 位验证码 | 验证通过进入系统 | | □ |

### 4.2 快速验证脚本（技术专家用）

```bash
# 登录获取 token
TOKEN=$(curl -s -X POST http://localhost:8000/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@ecodream.com","password":"admin123"}' \
  | python -c "import sys,json; print(json.load(sys.stdin).get('access_token',''))")

# 验证 token 有效
curl -s -H "Authorization: Bearer $TOKEN" http://localhost:8000/agents | head -c 50
```

## 五、L3 — 独立模块功能检查（测试专家主责）

**目标**：每个业务模块的 CRUD 操作独立完成，不依赖其他模块的数据。
**通过标准**：每个模块的「增/查/改/删」至少各执行一次成功。
**阻塞影响**：单个模块阻塞仅影响该模块及依赖它的 L4 流程。
**时间预算**：每页面约 10-15 分钟。

### 5.1 测试顺序（推荐优先级）

按「基础能力 → 内容生产 → 发布调度 → 数据洞察 → 系统管理」顺序：

```
1. Dashboard (首页数据加载)
2. AccountPool / Personas (账号管理)
3. ContentForge (内容创作)
4. Compliance / PlatformRules (合规检查)
5. AssetPool (素材库)
6. BrandKnowledge (品牌知识)
7. VetDrug (兽药库)
8. PersonaStory / StoryCockpit (故事线)
9. TaskHub / TaskHubCreate (任务中心)
10. ReviewPublishCenter (审核发布)
11. Publisher (发布调度)
12. LLMCockpit (LLM管理)
13. AgentOrchestra (Agent编排)
14. SkillHub (技能工坊)
15. CronCockpit (定时任务)
16. Timeline (时间线)
```

### 5.2 每个页面的通用检查清单

| # | 检查项 | 操作 | 预期结果 | 通过 |
|---|--------|------|----------|------|
| 1 | 页面加载 | 点击导航进入 | 无白屏/无报错/数据加载完成 | □ |
| 2 | 列表查询 | 进入页面 | 列表正确渲染（空数据时显示 EmptyState） | □ |
| 3 | 创建数据 | 点击新建 → 填写必填项 → 保存 | 保存成功，列表刷新出现新数据 | □ |
| 4 | 查看详情 | 点击某条数据 | 详情正确展示 | □ |
| 5 | 编辑数据 | 点击编辑 → 修改字段 → 保存 | 保存成功，数据已更新 | □ |
| 6 | 删除数据 | 点击删除 → 确认 | 删除成功，列表已移除 | □ |
| 7 | 筛选/搜索 | 输入关键词或使用筛选器 | 筛选结果正确 | □ |
| 8 | 分页（如有） | 切换分页 | 数据正确切换 | □ |
| 9 | 权限控制 | 用低权限账号访问 | 无权限操作被禁用/隐藏 | □ |
| 10 | 网络中断 | 断开网络 → 操作 → 恢复网络 | 有错误提示，不崩溃 | □ |

## 九、Bug 修复优先级与防反复策略

### 9.1 修复优先级

```
P0（阻塞）> L1/L2 缺陷 > L4 流程中断 > L3 独立模块缺陷 > L5 边界异常 > L6 UI优化 > P2
```

### 9.2 独立式优先原则

> **定义**：优先修复只影响单一模块、不依赖其他模块的 Bug。

| 类型 | 示例 | 修复策略 |
|------|------|----------|
| **独立式** | 某页面按钮点击无响应 | 直接修复，不等待其他模块 |
| **递进式** | 任务创建后列表不刷新 | 先修复列表刷新（L3），再验证任务创建（L4） |
| **阻塞式** | API 返回 500 导致多个页面白屏 | **必须先修复**，阻塞所有下游测试 |

### 9.3 防反复修复 checklist

开发修复 Bug 后，提交前必须自查：

```
□ 该修复是否只修改了目标文件？（避免连带修改）
□ 是否运行了该模块的单元测试？pytest tests/test_xxx.py -v
□ 是否运行了前端类型检查？cd apps/frontend && npx tsc -b --noEmit
□ 是否检查了对其他模块的调用影响？grep 搜索被修改的函数名
□ 是否更新了数据词典中对应的函数签名？
□ 是否在变更记录中登记了本次修复？
```

## 十一、快速启动命令

```bash
# 1. 启动全部服务（技术专家）
cd D:/project/EcoDreamOmni && docker-compose up -d --build

# 2. 验证环境（技术专家）
curl http://localhost:5173 && curl http://localhost:8000/health

# 3. 获取测试账号 Token（技术专家）
curl -X POST http://localhost:8000/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@ecodream.com","password":"admin123"}'

# 4. 运行后端测试（测试专家）
cd apps/backend && pytest -q --tb=short

# 5. 运行前端类型检查（测试专家）
cd apps/frontend && npx tsc -b --noEmit

# 6. 检查架构违规（测试专家）
cd apps/backend && pytest tests/test_no_direct_db_access.py -v
```
