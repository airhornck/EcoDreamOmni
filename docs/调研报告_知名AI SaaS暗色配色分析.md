# 知名 AI SaaS 产品暗色配色调研报告

> **调研范围**: Notion、Claude/Anthropic、ChatGPT/OpenAI、GitHub、Linear、Vercel、Cursor 等 7+ 款头部产品
> **调研日期**: 2026-06-03
> **目的**: 为 EcoDream Omni v4.0 暗色主题配色优化提供决策依据

---

## 一、各产品暗色配色拆解

### 1.1 Notion — 暖调柔和派

| 角色 | Hex | 说明 |
|------|-----|------|
| 画布背景 | `#191919` | 非纯黑，略带暖调 |
| 卡片表面 | `#252525` | 仅比背景高 3% 亮度，层次极 subtle |
| 主文字 | `#D4D4D4` | 偏灰白，非纯白，降低眩光 |
| 次要文字 | `#9B9B9B` | 中等灰 |
| 强调色 | `#865DBB`(紫) / `#447ACB`(蓝) | 饱和度偏低，柔和不刺眼 |
| 语义色 | `#4F9768`(绿) / `#CB7B37`(橙) / `#BE524B`(红) | 全部降饱和处理 |

**设计哲学**: "不打扰写作" — 低对比度、暖灰基底、色彩克制，让内容本身成为焦点。

---

### 1.2 Claude/Anthropic — 暖底多彩派

| 角色 | Hex | 说明 |
|------|-----|------|
| 画布背景 | `#0D0D0D` | 极暗，接近纯黑但非纯黑 |
| 代码背景 | `#30302E` | 暖 charcoal，带极微黄调 |
| 主文字 | `#FAF9F5` | **暖白**，非冷白，非常特别 |
| 强调色体系 | `#CC7BF4`(紫) / `#FBAD60`(橘) / `#F47B85`(玫) / `#9BE963`(绿) / `#5EEDED`(青) | 高饱和多彩，但仅在代码/AI输出中出现 |
| 品牌主色 | `#9D4EDD`(紫) / `#00A896`(青) / `#FFD23F`(金) | 紫+青+金三色品牌体系 |

**设计哲学**: "AI 是多彩的" — 日常界面极简，但 AI 生成内容（代码、思维链）用高饱和多彩表达智能的丰富性。**暖白文字**是其最大辨识度特征。

### 1.3 ChatGPT/OpenAI — 极致极简派

| 角色 | Hex | 说明 |
|------|-----|------|
| 画布背景 | `#080808` | 几乎纯黑 |

| 主文字 | `#ECECEC` | 近白 |
| 次要文字 | `#B4B4B4` | 浅灰 |
| 强调色 | **无** | 没有品牌强调色！仅靠语义灰度区分 |
| 用户气泡 | `#2F2F2F` | 深灰 |
| AI 气泡 | `#0D0D0D` | 与背景几乎融合 |

**设计哲学**: "让位于对话" — 界面完全隐身，没有任何视觉噪音。这是目前最极简的暗色方案。

### 1.7 Cursor — IDE 实用派

| 角色 | Hex | 说明 |

## 二、行业规律总结

### 2.1 背景色选择分布

```
极黑 (#000000~#0A0A0A):  Vercel、ChatGPT、Linear  → 3 家
深黑 (#0D0D0D~#121212):   GitHub、Claude            → 2 家
炭黑 (#191919~#2B2B2B):   Notion、Cursor/VS Code    → 2 家
```

**趋势**: AI 产品倾向**极黑/深黑**（#0A0A0A~#121212），非 AI 工具倾向**炭黑**（#1E1E1E+）。

### 2.2 文字色选择分布

| 策略 | 产品 | 效果 |
|------|------|------|
| 近白 `#ECECEC`~`#F5F5F5` | ChatGPT、Vercel、Linear | 清晰但长时间可能眩光 |
| 灰白 `#D4D4D4`~`#C9D1D9` | Notion、GitHub | 舒适，适合长文本阅读 |
| 暖白 `#FAF9F5` | Claude | 独特、温暖、降低冷感 |

### 2.3 强调色策略分布

| 策略 | 产品 | 适用场景 |
|------|------|---------|
| 无强调色（纯灰度） | ChatGPT | 对话型产品 |
| 单一强调色 | Vercel（蓝）、Linear（靛蓝紫） | 工具型产品 |
| 柔和多色 | Notion | 内容/文档型产品 |
| 高饱和多彩 | Claude | AI 内容展示型产品 |
| 蓝调单色系 | GitHub | 工程/开发者产品 |

### 2.4 表面层级策略

| 产品 | 层级数 | 层级差距 |
|------|--------|---------|
| ChatGPT | 2 | ~6% |
| Linear | 2 | ~6% |
| Vercel | 2~3 | ~7% |
| GitHub | 3~4 | ~8% |
| Notion | 3~4 | ~3%（极 subtle） |
| Claude | 2 | ~20%（代码区明显分离） |

## 三、当前 EcoDream Omni 配色诊断

### 3.1 现有配色回顾

| 角色 | 当前值 | 问题分析 |
|------|--------|---------|
| 背景 | `#0A0B0F` | ✅ 符合行业标准（Rich Black） |
| 主画布 | `#0F1117` | ✅ 合理抬升 |
| 卡片 | `#161B22` | ✅ 与 GitHub 一致 |
| 主文字 | `#E8E8ED` | ⚠️ 偏冷、偏亮，接近 ChatGPT 的刺眼风格 |
| 次要文字 | `#A1A1AA` | ✅ 合理 |
| 辅助文字 | `#71717A` | ⚠️ 在 `#0A0B0F` 上对比度约 5.2:1，刚好过线 |
| AI 强调色 | `#A371F7` | 🔴 **问题最大**：紫偏红、饱和度偏高、在暗底上发闷 |
| 成功 | `#3FB950` | ✅ 标准 |
| 信息 | `#58A6FF` | ✅ 与 GitHub 一致 |
| 警告 | `#D29922` | ✅ 标准 |
| 危险 | `#F85149` | ✅ 与 GitHub 一致 |
| 边框 | `rgba(255,255,255,0.10)` | ⚠️ 偏暗，部分场景几乎不可见 |

### 3.2 核心问题

1. **AI 强调色 `#A371F7` 不够高级**
   - 与 Claude 的 `#CC7BF4` 相比，明度偏低、在暗底上"发灰"
   - 与 Linear 的 `#5E6AD2` 相比，饱和度过高、不够克制
   - 与 Vercel 的 `#0070F3` 相比，缺乏"科技感"

2. **文字偏冷**
   - `#E8E8ED` 是冷白，没有 Claude `#FAF9F5` 的温暖感
   - 长时间使用会产生"手术室"般的冷峻感

3. **表面层级过密**
   - 5 层表面（#0A0B0F → #0F1117 → #13151C → #161B22 → #181A23）
   - 每层差距仅 2~3%，视觉上难以区分
   - Notion 仅 3 层，GitHub 仅 2~3 层

4. **边框过暗**
   - 10% 白透明度在 Rich Black 上几乎隐形
   - GitHub 用 `#30363D`（约 18% 亮度），明显更可见

## 五、决策建议

### 5.1 推荐方案 A（Claude 暖白派）

**理由**：
1. **差异化**: 暖白文字在中文 AI 产品中几乎独一无二，Claude 是极少数采用者
2. **产品定位匹配**: EcoDream Omni 是"AI 内容生产平台"，需要长时间阅读和编辑，暖白比冷白更舒适
3. **AI 强调色**: `#CC7BF4` 比当前 `#A371F7` 在暗底上更通透、更"发光"，传递 AI 的智能感
4. **边框优化**: `#27272A` 比 `rgba(255,255,255,0.10)` 更明确，UI 边界更清晰

### 5.2 如果强调"开发者工具"属性 → 选方案 B

如果产品的核心用户是开发者（而非运营/内容创作者），GitHub 蓝调方案会让他们感到"回家"。

### 5.3 如果强调"高级感/极简" → 选方案 C

如果产品定位偏"高端 SaaS"、"企业级"，Linear 式的极暗克制会更显品质。
