# 需求对齐约束 — 通用提示词（精简版 v4.0）

> **用途**：粘贴于每段具体需求末尾，作为 Agent 执行的强制前置检查。
> **完整版真源**：`docs/需求对齐约束_通用提示词_v4.0.md`

---

在处理上述需求前，必须执行以下七项强制检查。

**任一项无法通过 → 暂停实施，向用户说明卡点，等待澄清后再继续。**

---

## 【1】PRD 真源对齐

- PRD v4.0 (`EcoDream_Omni_PRD_v4_AI_Native_Architecture.md`) 有定义？**无 → 补充 PRD + 更新 `PRD偏差报告_v4.0.md`**
- 触发专家评审？（新增页面/模型/API Router/外部依赖/Mode C 变更/跨 Phase）**是 → 4 维度评审，综合评分 ≥ 3.0 方可实施**
- 超出当前 Phase？**是 → 仅做预留接口，不实现完整逻辑**

## 【2】技术架构对齐

- 仅维护现有调用关系（函数体/参数/返回值）？**涉及架构调整（Router/Service/ORM/前缀/DB字段/依赖）→ 暂停，提供影响评估，等待用户批准**
- Agent 是否直接 `import AsyncSession` 或 Agent 间直接调用？**是 → 禁止；DB 走 Function 层，通信走 EventBus**

## 【3】前端 Mode C 合规（如涉前端）

- 工作区出现业务按钮（保存/生成/删除/导出/上传按钮）？**是 → 禁止；唯一入口为 Copilot Action Cards**
- 新增实验室能力？**是 → `labStore.ts` 注册 `LAB_CAPABILITIES`，画布放 `components/lab/`，API 独立**

## 【4】HTML 预览审核（如涉前端页面）

- 新增/重构 Page 组件或修改全局布局/设计 Token？**是 → 生成 HTML 预览 (`demo/page-preview/`)**
- 自检查：Mode C 合规 + 颜色仅用设计系统 Token + 三栏布局正确 + Copilot 状态机正确
- **审核通过后方可同步到 React 代码**

## 【5】文档连锁更新

- 新 API → `数据词典_v4.0/02-后端API路由层.md` + `契约与数据/01-API接口契约.md` + `数据词典_v4.0/05-模块调用关系矩阵.md`
- 新 ORM → `数据词典_v4.0/01-数据库模型层.md` + `契约与数据/02-数据库ER图.md`
- 新 Store/路由 → `数据词典_v4.0/04-前端Store与路由.md` + `05-模块调用关系矩阵.md`
- 任何文档变更 → `文档总纲_v4.0.md` 索引同步

## 【6】变更记录登记

- 输出到 `docs/变更记录_v4.0/YYYY-MM-DD/开发变更总结_YYYY-MM-DD.md`
- 必填：需求概述 / PRD 对齐 / 修改文件清单 / 调用关系变更 / 测试验证 / 文档更新 / 风险与后续
- 触发专家评审 → 附加 `需求名_专家评审报告.md`（4 维度评分 + unanimous 共识）

## 【7】工程纪律

- **Red-Green-Blue**：🔴 先写失败测试 → 🟢 最小实现 → 🔵 重构
- **质量门禁**：`tsc --noEmit` 0 errors + `eslint` 0 errors + `pytest` ≥ 80%
- **LLM 优先**：分析类需求主路径用 LLM，规则仅校准（关键词库 50-100 MVP）
- **禁止**：不写测试 / 跳过红灯 / 一次改多模块 / 删他人测试 / 不更新文档 / Agent 直连 DB / Agent 间直接调用 / 绕过 HTML 预览审核 / 评分 < 3.0 未经修改直接实施
