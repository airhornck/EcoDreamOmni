## 一、产品定位与价值主张

### 1.1 一句话定位

> **面向宠物健康品牌市场团队的「素人号矩阵AI内容生产与分发平台」**——AI负责内容生成与流量预测，技术对抗层负责账号安全，运营负责策略与审核。

### 1.2 核心价值主张

| 价值维度 | 传统方式 | 本平台方案 | 量化指标 |
|----------|----------|------------|----------|
| **内容产能** | 1个运营日产3-5篇 | AI辅助生成+自动化分发，1个运营管理50个账号，日产50篇 | 内容产能提升10倍 |

- 每个账号绑定**1个人设**

### 4.2 Layer 1: 基础设施层

#### 4.2.1 指纹信誉系统

**流程**：

```
新指纹模板生成（从真实指纹库采样）
    │
    ▼
一致性校验（时区↔IP城市、语言↔字体等）
    │
    ▼
测试期（7天，分配1个测试账号运行）
    │
    ├─ 无异常（无验证码、无限流、无封号） → 标记"可信"，分配给正式账号
    ├─ 轻微异常（1次验证码） → 延长测试期至14天
    └─ 严重异常（限流/封号） → 标记"高风险"，废弃该指纹模板
    │
    ▼
定期复评（每月重新评估所有"可信"指纹，异常率>5%的降级为"观察"）
```

**指纹库规模**：维护10万+真实浏览器指纹作为采样种子，每月更新。

#### 4.2.2 IP信誉系统

**流程**：与指纹信誉系统类似，新IP需经过7天测试期。

**IP池管理**：
- 合作3-5家住宅代理服务商，确保IP来源多样化。
- IP按城市/ISP分组，建立IP-城市映射表。
- 每个IP最多绑定2个活跃账号。
- IP切换策略：正常期固定IP，异常期切换备用IP。

### 4.3 Layer 2: 身份隔离层

#### 4.3.1 设备指纹差异化（完整保留）

**指纹维度**（完整列表）：

| 维度 | 配置项 | 差异化策略 |
|------|--------|-----------|
| 浏览器 | Chrome/Safari/Firefox/Edge | 按设备类型分配 |
| User-Agent | 完整UA字符串 | 从真实UA库采样 |
| 屏幕分辨率 | 390x844 / 414x896 / 375x812等 | 匹配设备类型 |
| 颜色深度 | 24/32 | 随机 |
| 时区 | Asia/Shanghai / Asia/Beijing等 | 匹配IP城市 |
| 语言 | zh-CN / zh-TW / en-US等 | 匹配地区 |
| 平台 | iPhone / Android / Win32 / MacIntel | 匹配设备类型 |
| Canvas哈希 | 2D渲染指纹 | 加入随机噪声 |
| WebGL | Vendor/Renderer/参数 | 匹配设备GPU |
| 字体列表 | 系统字体 | 从真实字体库采样 |
| Audio指纹 | AudioContext输出 | 加入微小扰动 |
| WebRTC | IP泄露控制 | 关闭或配置为代理IP |

#### 4.3.2 IP代理池（完整保留）

| 策略 | 配置 |
|------|------|
| 代理类型 | 住宅静态代理（Residential Static Proxy） |
| IP来源 | 3-5家服务商，覆盖一线至三线城市 |
| 账号/IP比例 | ≤2账号/IP |
| 切换策略 | 固定为主，异常时切换 |
| 测试期 | 7天测试，无异常后正式使用 |

#### 4.3.3 关联隔离

**四维隔离策略**：

| 维度 | 隔离规则 |
|------|----------|
| **设备** | 每个账号独立浏览器Profile，指纹不重复 |
| **IP** | 同IP≤2账号，IP城市与内容地域一致 |
| **时段** | 账号分组（早/中/晚），同组登录间隔≥15分钟 |
| **内容** | 同Brief分配给不同素人时，生成差异化内容（相似度<30%） |

### 4.4 Layer 3: 行为安全层

#### 4.4.1 行为仿真（完整保留+优化）

**行为模式库**：从真实用户录屏中提取以下模式：

| 行为类型 | 仿真策略 |
|----------|----------|
| **鼠标移动** | 贝塞尔曲线轨迹，速度先加速后减速 |
| **点击** | 目标区域随机偏移5-15像素，点击前有0.2-0.5秒停留 |
| **打字** | 高斯分布间隔（均值120ms，标准差40ms），偶尔退格、暂停 |
| **滚动** | 物理惯性模型：加速→匀速→减速→停顿→继续 |
| **阅读** | 页面停留时间按内容长度正比分布（100字/秒） |
| **离开** | 操作序列中插入随机时长（30-300秒）的"离开"时段 |

**个性化行为画像**：

每个账号配置独立的行为参数：

```json
{
  "behavior_profile": {
    "typing_speed_mean": 120,
    "typing_speed_std": 40,
    "scroll_acceleration": 0.8,
    "scroll_pause_frequency": 0.3,
    "click_precision": 0.85,
    "session_duration_min": 300,
    "session_duration_max": 1800,
    "active_hours": [8, 12, 20],
    "active_variance": 30
  }
}
```

#### 4.4.2 频率控制与错峰调度

| 账号类型 | 日浏览 | 日点赞 | 日评论 | 日发布 | 登录次数 |
|----------|--------|--------|--------|--------|----------|
| 新号（<30天） | ≤30 | ≤10 | ≤5 | ≤1 | ≤2 |
| 成长期（30-90天） | ≤50 | ≤15 | ≤8 | ≤1 | ≤2 |
| 老号（>90天） | ≤80 | ≤20 | ≤10 | ≤2 | ≤3 |

**错峰策略**：
- A组：早8-10点
- B组：午12-14点
- C组：晚18-22点
- 同组账号登录间隔≥15分钟

#### 4.4.3 异常熔断

| 异常类型 | 触发条件 | 处置 | 恢复条件 |
|----------|----------|------|----------|
| 验证码 | 触发图形/短信验证码 | 暂停操作24h | 24h后自动恢复 |
| 限流 | 操作被拒绝/频率限制 | 暂停操作48h | 48h后测试恢复 |
| 登录失败 | 连续2次登录失败 | 暂停操作72h | 人工检查 |
| 内容下架 | 单篇内容被平台下架 | 标记该内容模板为高风险 | 分析原因后恢复 |
| 账号警告 | 收到平台违规警告 | 暂停发布，仅保留浏览 | 7天后测试恢复 |

#### 6.2.1 工程可实现性（结构闭环与反馈闭环）

- **结构侧**：「结构预测」指 **模板/字段符合度 + 可选 LLM rubric 评分**，**不等于**预测平台排序或流量；**调整闭环**为运营采纳建议后**显式触发**再生成（ContentForge）并再次过合规。**禁止**将「自动循环改写直至预测区间收窄」列为 MVP 必达能力。

- **互动区间**：冷启动为 **宽先验**；UI 区分 `interval_mode=prior|fitted`。对运营展示的「+x% 互动」须来自 **A/B 或随机实验**，否则仅能使用**启发式清单**（非因果承诺）。校准任务为 **批处理/异步**，与在线推理解耦。

#### 8.1.2 防封控策略体系（完整保留+优化）

| 策略层 | 机制 | 实现细节 | PlatformRule注入 |
|--------|------|----------|------------------|
| **设备隔离** | 每个账号独立浏览器Profile | Playwright独立Context，指纹来自**真实采样库**，Canvas/WebGL/Fonts/Timezone/WebRTC全指纹差异化，一致性校验引擎确保各维度匹配 | 同设备最多1个活跃账号 |
| **IP代理** | 住宅IP池轮换 | 静态住宅代理，按城市/ISP分组，同IP下账号数≤2，**IP信誉系统**自动淘汰被标记IP，异常时切换备用IP | 同IP登录间隔≥30分钟 |
| **登录模拟** | 人类行为仿真 | 随机延迟5-30秒、模拟打字速度（高斯分布，均值120ms）、**贝塞尔曲线点击轨迹**、**物理惯性滚动**、随机"离开"时段 | 单日登录≤2次，失败≥2次冷却24h |
| **操作限速** | 行为频率控制 | 新号（<30天）：浏览≤30次/日、点赞≤10次/日、评论≤5次/日、发布≤1次/日；老号（>30天）：发布≤2次/日 | 新号/老号操作阈值差异化 |
| **错峰调度** | 登录时间分散 | 账号分组（A组早8-10点、B组午12-14点、C组晚20-22点），避免并发登录 | 同组账号登录间隔≥15分钟 |
| **异常熔断** | 实时风控 | 触发验证码/限流/登录失败时，自动切换账号状态为`warming`，暂停操作24-72h | 异常等级与冷却时长映射 |
| **关联隔离** | 四维隔离 | 设备+IP+时段+内容相似度<30%（通过文本指纹计算） | 矩阵账号强制隔离 |

#### 8.1.3 账号健康度评分算法

$$\text{HealthScore} = 0.25 \times \text{LoginSuccess} + 0.25 \times \text{ComplianceRate} + 0.20 \times (100 - \text{LimitationFlag}) + 0.15 \times \text{Naturalness} + 0.15 \times \text{AssociationRisk}$$

**状态转换规则**：

| 当前状态 | 触发条件 | 新状态 | 动作 |
|----------|----------|--------|------|
| active | health_score < 60 | warming | 暂停发布，仅保留浏览 |
| active | 平台限流/验证码 | warming | 冷却24h |
| warming | 冷却期满 + health_score ≥ 70 | active | 恢复操作 |
| warming | 连续2次登录失败 | restricted | 人工介入 |
| restricted | 人工审核通过 | recycled | 解绑人设，归档账号 |
| any | 平台封禁 | banned | 人设转移，账号废弃 |

### 8.3 PlatformRule Engine（平台规则中枢）【完整保留+强化】

#### 8.3.1 双轨合规体系

| 层级 | 规则类型 | 更新频率 | 示例 |
|------|----------|----------|------|
| **L1 法律红线** | 广告法、兽药法、医疗广告限制 | 季度 | 禁止处方药、禁止诊疗建议、必须标注合作 |
| **L2 平台规则** | 社区规范、算法机制、流量规则 | 月度 | 发布频率限制、标签规范、广告标注要求 |

| **L4 动态风控规则** | 平台临时风控、节日/大促策略 | 实时 | 618期间商业笔记限流、某类关键词降权 |

### 8.4 TrendScout（趋势侦察）【完整保留+优化】

**核心职责**：爬取热点、提取特征、输出阶段化趋势报告

**爬取策略（优化版）**：

| 维度 | 配置 |
|------|------|
| **抓取范围** | 关键词搜索结果TOP15、话题页热门 |
| **过滤条件** | 排除医疗认证号、品牌官方号、粉丝>10万的大号 |
| **保留条件** | 素人感强（粉丝<1万、内容生活化、互动率高） |
| **特征提取** | 标题结构、正文逻辑、标签组合、发布时段、互动数据 |
| **频率控制** | 单IP单账号日爬取≤3次，间隔≥10分钟 |
| **范围限制** | 仅爬取**公开可见数据**，不爬取私信、收藏等非公开数据 |

### 8.5 ContentForge（内容锻造）【完整保留】

**核心职责**：基于人设Voice+方法论模板+趋势报告，生成差异化文案

**生成策略**：

| 策略 | 说明 |
|------|------|
| **Voice注入** | 每篇内容强制携带人设的catchphrases、emoji风格、句式长度 |
| **阶段模板** | 按A/I/P/L阶段调用对应结构模板 |
| **差异化生成** | 同一选题生成15篇，通过标题变体、情感倾向、场景选择、CTA方式差异化 |
| **素人感强化** | 模拟真实用户的不完美：偶尔错别字（1-2个/篇）、口语停顿 |
| **合规前置** | Prompt中注入L1规则，生成即合规 |

### 8.6 ComplianceGuard（合规守卫）【完整保留+强化】

**检测流程**：

```
输入：候选文案
    │
    ▼
[L1 法律红线检测] ──► 正则匹配+语义理解 ──► 命中？──► 是 → 硬拦截+替换建议
    │                                           └──► 否 → 继续
    ▼
[L2 平台规则检测] ──► 规则引擎+模型分类 ──► 命中？──► 是 → 黄标预警+优化建议
    │                                           └──► 否 → 通过
    ▼
[L3 行业合规检测] ──► 宠物医药专用规则 ──► 命中？──► 是 → 拦截+修改建议
    │                                           └──► 否 → 通过
    ▼
输出：合规报告（风险分0-100，拦截/通过/建议修改）
```

**证据链留存**：每篇内容合规审核记录永久存档（留存≥2年）。

### 8.8 PoolPredictor（流量预测）【重新定义】

**核心职责**：提取内容特征→预测互动量区间→输出优化建议

> **工程边界**：优化建议仅为**启发式**（标题/标签/时段等），**不得**包装为未经 A/B 验证的因果增幅；小样本下 MAPE 数字为**经验区间**非合同 SLA。与「内容结构是否合格」（方法论模板）**解耦**——结构合规不保证互动区间上移。

**预测目标修正**：

| 预测目标 | 原方案 | 修正后 | 原因 |
|----------|--------|--------|------|
| 流量池层级 | L0-L5 | **废弃** | 平台未公开定义 |
| CES得分 | 精确值 | **互动量区间**（点赞/评论/收藏） | 基于历史数据可预测 |
| 相对排序 | 无 | **A/B内容排序预测** | 可行 |

**模型演进路径（修正版）**：

| 阶段 | 数据量 | 模型 | 输出 | 精度预期 |
|------|--------|------|------|----------|
| 冷启动 | 0-20篇 | 信息先验+同类基准 | 基准区间+方向性建议 | MAPE 常高；**仅作参考** |
| 探索期 | 20-50篇 | 贝叶斯线性回归/随机森林 | 互动量区间预测 | MAPE 随标签质量下降 |
| 稳定期 | 50-200篇 | XGBoost/梯度提升 | 互动量点预测+区间 | 须持续监控漂移 |
| 精细化 | 200+篇 | 集成模型+SHAP（可选） | 预测+归因+优化建议 | SHAP 为解释性辅助，非因果 |

**特征提取（18维修正版）**：

| 维度 | 特征 | 类型 | 可获取性 |
|------|------|------|----------|
| 标题 | 字数、含数字/emoji、疑问/感叹语气、关键词匹配度 | 连续/二元 | ✅ |
| 正文 | 字数、段落数、信息密度、情感倾向、实用价值 | 连续/二元 | ✅ |
| 标签 | 数量、热度均值、精准度 | 连续 | ✅ |
| 账号 | 账号年龄、粉丝数、历史平均互动、近7天活跃度 | 连续 | ✅ |
| 时间 | 时段、星期、节假日因子 | 连续/分类 | ✅ |
| 人设 | Voice匹配度、内容类型 | 连续/分类 | ✅ |
| 合规 | 合规风险分 | 连续 | ✅ |

#### 9.2.3 调用策略

**负载均衡**：
- 按Token消耗速率进行负载均衡，避免单模型API限流。
- 多模型并行调用同一任务，取最优结果（适用于ContentForge内容生成）。

**成本优化**：
- 简单任务（格式化、摘要）路由至低成本模型（GPT-4o-mini）。
- 复杂任务（创意生成、推理）路由至高能力模型（GPT-4o/Claude 3.5）。
- 缓存层：相同Prompt的响应缓存30分钟，命中率可达40%。

**故障降级**：
- 主模型超时（>10秒）→ 自动切换备用模型。
- 主模型API报错（429/500/503）→ 指数退避重试3次 → 切换备用模型。
- 所有外部模型不可用时 → 切换至本地私有化模型（Qwen2-72B）。

#### 9.5.2 成本控制策略

1. **Prompt压缩**：使用语义压缩技术，将长Prompt压缩为关键信息，减少Input Token 20-30%。
2. **响应截断**：设置Max Tokens上限，避免模型生成冗余内容。
3. **批量调用**：ContentForge同时生成15篇内容时，使用批量API调用降低单位成本。
4. **缓存复用**：相同或相似Prompt的响应缓存30分钟，命中率40%。
5. **模型降级**：低复杂度任务自动降级至低成本模型（GPT-4o-mini）。

### 9.6 安全与合规

#### 9.6.1 数据出境合规

| 场景 | 处理方式 | 合规依据 |
|------|----------|----------|
| 使用境外LLM（OpenAI/Anthropic） | 数据脱敏后传输（去除品牌名、账号信息、个人隐私） | 《数据出境安全评估办法》 |

| 合规检测 | 优先使用国内LLM，确保法规理解准确 | 法规语境本土化 |

#### 9.6.2 Prompt注入防护

| 攻击类型 | 防护措施 |
|----------|----------|
| **Prompt Injection** | 用户输入经过转义和过滤，与系统Prompt隔离 |
| **Jailbreak** | 系统Prompt中嵌入"拒绝执行违规指令"的强化约束 |
| **数据泄露** | 禁止LLM在响应中输出系统Prompt内容 |
| **幻觉** | 关键事实（法规条文、产品信息）通过RAG从知识库检索，不依赖模型记忆 |

#### 9.6.3 审计与监控

- **Prompt日志**：所有调用的Prompt和Response记录留存≥90天。
- **敏感词审计**：Prompt和Response中的敏感词（如处方药名）触发告警。
- **成本监控**：单账号日Token消耗上限，超限自动告警。
- **质量监控**：LLM输出质量评分，低于阈值触发模型切换。

## 十、SkillHub技能中枢体系

> **本章定位**：系统的"知识引擎"，负责管理Agent可复用的能力单元（Skill）。Skill是Agent执行特定任务的标准化指令包，包含Prompt模板、领域知识、执行脚本和记忆片段。

---

### 10.1 技能标准规范

#### 10.1.1 技能定义

**Skill** = Agent可复用的能力单元，包含：
- **Metadata**：技能名称、描述、版本、标签、适用Agent
- **Instruction**：核心SOP（标准操作流程），即Prompt模板
- **Reference**：领域知识（法规、案例、数据），条件触发加载
- **Script**：确定性执行脚本（Python/Shell），执行时不读入上下文
- **Memory**：该Skill的执行记忆片段（成功案例、失败案例）

#### 10.1.2 技能四层加载架构

采用**渐进式披露架构**，按需加载，控制Token消耗：

```
skills/
└── {skill_name}/
    ├── SKILL.md              # L1 Metadata + 核心Instruction
    ├── instructions/
    │   └── detail.md         # L2 扩展指令（按需加载）
    ├── references/
    │   ├── compliance_lib.md # L3 领域知识（条件触发）
    │   └── case_studies/     # L3 案例库
    ├── scripts/
    │   └── execute.py        # L4 确定性脚本（执行不读入上下文）
    └── memory/
        ├── episodic/         # 该Skill的执行记忆片段
        └── success_patterns/ # 提取的成功模式
```

**四层加载策略**：

| 层级 | 内容 | 加载策略 | Token权重 |
|------|------|----------|-----------|
| **L1** | Metadata（名称、描述、版本、标签） | Always-On | <1% |
| **L2** | Instruction（核心SOP） | On-Demand（Agent启动时加载） | 5-10% |
| **L3** | Reference（法规、案例、数据） | Context-Triggered（命中关键词时加载） | 可变 |
| **L4** | Script（Python/Shell） | Execution-Only（执行时调用，不读入LLM上下文） | 0 |

#### 10.2.2 Evolved Skills（系统进化）

**生成条件**（SkillSmith自动提取）：
- 某内容策略连续3次互动量超过该账号历史均值50%，或
- 某标题公式在5篇中4篇达到预测区间上限，或
- 某标签组合平均互动量超过同品类均值30%

**进化流程**：

```
高表现内容识别（DataAnalyst标记）
    │
    ▼
SkillSmith提取成功模式（标题结构、正文结构、标签组合、发布时段）
    │
    ▼
生成Evolved Skill草案
    │
    ▼
人工审核（运营确认该模式可复用）
    │
    ├─ 不通过 → 丢弃
    └─ 通过 → 标记为"待验证"
    │
    ▼
小范围验证（5-10个账号试用，运行1周）
    │
    ├─ 验证失败（成功率<60%） → 回滚
    └─ 验证成功（成功率≥60%） → 正式发布为Evolved Skill
    │
    ▼
全局推广（所有适用账号启用该Skill）
```

## 十二、流量预测模型设计（重新定义）

### 10.1 模型假设修正

**原方案假设（修正前）**：
- 假设可以预测平台内部的"流量池层级"（L0-L5）
- 假设可以精确预测CES得分
- 假设冷启动期即可达到75%准确率

**修正后假设**：
- 平台流量池层级为黑盒，不可预测
- 可预测的是基于历史数据的**互动量区间**（点赞/评论/收藏）
- 预测精度随数据量积累逐步提升
- 初期MAPE 50-80%，稳定期MAPE 20-35%

### 10.2 冷启动期：信息先验与基准

**信息先验初始化**：

$$\theta_{\text{账号}} \sim \text{Beta}(\alpha_0, \beta_0)$$

$$\alpha_0 = 1 + \frac{1}{N}\sum_{i \in \text{同类账号}} \frac{\text{互动量}_i}{50}$$

$$\beta_0 = 1 + \frac{1}{N}\sum_{i \in \text{同类账号}} \left(4 - \frac{\text{互动量}_i}{50}\right)$$

**输出**：基准区间+方向性建议（如"该选题在同类账号中平均互动量为30-60"）

### 10.3 探索-利用期：贝叶斯线性回归

**模型**：

$$\text{互动量} = \beta_0 + \sum_{i=1}^{18} \beta_i X_i + \varepsilon, \quad \varepsilon \sim \mathcal{N}(0, \sigma^2)$$

**输出**：互动量区间预测（如"点赞25-60，中位数42"）

### 10.4 稳定期：集成模型

**集成策略**：

$$\text{FinalPrediction} = 0.5 \times \text{XGBoost} + 0.3 \times \text{RandomForest} + 0.2 \times \text{BayesianLinear}$$

**SHAP可解释性**：输出每篇内容的特征贡献度。

### 10.5 时段竞争优化：ARIMA

**模型**：

$$y_t = c + \phi_1 y_{t-1} + \phi_2 y_{t-2} + \theta_1 \varepsilon_{t-1} + \varepsilon_t$$

其中$y_t$为$t$时段的宠物健康类内容平均互动量。

**输出**：未来7天各时段竞争系数+推荐窗口。
