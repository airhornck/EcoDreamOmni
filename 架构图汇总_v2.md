## 4. 业务流程图

### 4.1 V1.0 广告植入主流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    V1.0 广告植入制作流程                                    │
└─────────────────────────────────────────────────────────────────────────────┘

[开始]
  │
  ▼
[创建项目 - 选择"短视频广告"]
  │
  ▼
[选择广告类型]
  ├── 硬植入（产品+AI场景）
  ├── 软植入（产品+真实视频）
  ├── 口播（数字人介绍产品）
  └── 混合（多种组合）
  │
  ▼
[上传产品素材]
  ├── 产品图（PNG透明底）
  ├── 产品视频（360°展示）
  └── 品牌素材（Logo/Slogan）
  │
  ▼
[内容创作]
  ├── 硬植入：选择场景 + 调整光照/角度
  ├── 软植入：上传视频 + 选择植入点 + 跟踪校准
  ├── 口播：输入文案 + 选择数字人 + 调整语气
  └── 混合：时间轴编排各片段
  │
  ▼
[生成视频]
  ├── 提交生成任务
  ├── 异步处理（队列调度）
  └── 实时进度推送
  │
  ▼
[后期处理]
  ├── 自动剪辑
  ├── 配音配乐
  ├── 字幕生成
  └── 品牌标识添加
  │
  ▼
[多平台导出]
  ├── 抖音版（9:16）
  ├── 快手版（9:16）
  ├── TikTok版（9:16 + 多语言）
  └── 信息流版（前3秒优化）
  │
  ▼
[发布投放]
  ├── 下载视频
  ├── API直发抖音
  └── 数据追踪
  │
  ▼
[结束]
```

## 10. 关键设计决策

| 决策点 | V1.0方案 | V2.0扩展方案 | 理由 |
|--------|---------|-------------|------|
| Project类型 | type字段区分 | 共用一套模型 | 便于演进，减少迁移成本 |
| 场景管理 | 独立场景库 | 增加consistency_group_id | V2.0支持场景一致性 |
| 演员管理 | 基础数字人 | 扩展actor_profile字段 | V2.0支持数字演员系统 |
| 视频生成 | 第三方API | 混合（API+自建集群） | V1.0快速验证，V2.0降成本 |
| 数据库 | PostgreSQL | +MongoDB（剧本/分镜） | V2.0文档型数据更适合 |
| 消息队列 | RabbitMQ | +Kafka（大数据量） | V2.0批量生成需要 |
